## 0. 编制说明与角色定位

本方案由**小酷**主理编制。三角色分工严格遵循 `docs/roles/`：

| 角色 | 系统身份 | 在本方案中的职责 |
|---|---|---|
| **小酷** | WorkBuddy 会话 | 需求澄清、设计方案、写完整任务提示词、审核结果、协调发布、向胡子老师交付 |
| **小M** | mavis / Mavis | 独立只读复审，P 级强制，输出「通过 / 有条件通过 / 不通过」+ 证据 |
| **小猫** | coder / Coder | 边界明确的机械执行（文档同步、图标替换、批量收敛、跑测试） |

**胡子老师**是唯一业务拍板人、上岗批准人、真实环境验收人。

### 执行说明

本 WorkBuddy 会话中，我以**小酷**岗位履职：设计方案 + 写完整提示词 + 审核小猫/小M 结果。

小猫（执行）和小M（复审）是**独立外部AI系统**。任务派发方式：
- 小酷在本会话中写出完整任务提示词（自包含、小猫/小M 拿到即能执行）
- 胡子老师将提示词复制发给小猫/小M 的独立会话
- 小猫/小M 产出后，胡子老师将结果回传小酷审核

**小酷不代执行，不模拟小猫/小M 的产出**——这是岗位铁律。S 级小修（< 20 行无业务影响）和文档同步属于小酷可直接处理的范围。

### 任务分级标准（S/A/P）
- **S 级**：< 20 行且无业务影响 → 小酷自审后直接提交发布。
- **A 级**：> 20 行或影响业务逻辑 → 设计 → TDD → 小酷自审 → 小M 复审 → 验收。
- **P 级**：涉及资金 / 权限 / 数据库结构 → 设计 → TDD → 小酷自审 → **小M 提交前复审（强制）** → **胡子老师关键链路验收** → 提交推送 → 部署核验。

## 1. 现状盘点（精炼 + 已独立验证）

**整体结论**：核心 MLM 链路（注册 → 安置 → 升级 → 下单 → 奖励 → 积分释放 → 提现 → 退款 → 分红 → 通知）均已打通，前后端基本成对，无空壳页面。测试 47 个文件、覆盖率约 85%（语句 85.4% / 条件 78.4% / 方法 83%）。无硬编码密钥。Supabase Data API 已封锁生产业务表直访。

**已独立验证的三处关键事实**（非照抄排查结论）：
1. `AdminService.settleDividends()`（`src/lib/services/admin.service.ts:258`）全库**零业务调用**，仅被 `__tests__/services/admin.test.ts` 引用 → 死代码 + 重复发放地雷。
2. `computeMaxLayers()`（`reward.service.ts:44-54`）对会员返回 10 层、0 直推经销商的经销商返回 2 层，与 PRD §2.4.2「会员无资格、0 直推=0 层」冲突。
3. `middleware.ts:88-91` 仅拦截 `/api/admin/*`，用户侧接口第一道无 middleware 保护，依赖各路由内 `verifyPermission` 兜底。

## 2. 三角色协同工作流

```
胡子老师（拍板 / 批准 / 验收）
   │ 委托
   ▼
小酷：澄清 → 分级(S/A/P) → 写设计+计划（A/P级先批）
   ├─ 派 小猫：docs/roles/tasks/xiaomao/todo/小猫_NNN号任务.md（六要素齐备）
   │       → 小猫机械执行 → 结果回小酷
   └─ 派 小M：docs/roles/tasks/xiaom/todo/小M_NNN号任务.md（基线/范围/验证要求）
           → 小M 只读复审 → 三态结论+证据回小酷
   │
   ▼ 汇总证据
小酷：精确暂存（禁 git add . / -A）→ push → 核对 git log origin/main ──► 胡子老师验收
```

**派单六要素**（小猫任务必须齐备，缺一项即停）：目标 / 基线 / 范围 / 禁止操作 / 验证命令 / 完成标准。
**复审强制**：P 级必须小M 提交前复审；A 级建议小M；S 级小酷自审。
**发布铁律**：`commit/push ≠ 部署完成`（push 后 `git log origin/main --oneline -1` 核对 + Vercel Ready 核对）；UI 改动须真实截图（铁律 1/2）。
**代码铁律**：service 改完须 `grep` 所有真实调用入口（铁律 11）；admin 状态变更须自动通知用户（铁律 12）；字段名以 `schema.prisma` 为准（铁律 4）。
