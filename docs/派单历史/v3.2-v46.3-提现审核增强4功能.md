## 已知 trade-off（Mavis 决策记录）

1. **批量审核非真正事务**——`batchReview` 是 for 循环，每条独立事务，失败 1 条不影响其他——简化实现，避免大事务锁等待
2. **通知模型简化**——站内信是简化版，**不**支持推送/邮件/短信——**未来** v46.x 扩展
3. **审计日志关联**——admin 用户如果删除，auditLog 仍保留（admin select 显示）——**已知 trade-off**
4. **migration 数据兼容**——3 新 model + Withdrawal/User 加字段，已有数据需要默认值（`reviewedBy`/`rejectTemplateId`/`remark` 都 nullable）
5. **coverage buffer 收窄**——4 个新 service 加，buffer 1.85% → 0.85-1.35%——下次 commit 必带 coverage 数字
6. **不动 4 字段体系**——withdrawal 路径继续用 balance/frozenBalance，**不**走 earningsAvailable（v43-7 Batch 2.2 已知 trade-off）
