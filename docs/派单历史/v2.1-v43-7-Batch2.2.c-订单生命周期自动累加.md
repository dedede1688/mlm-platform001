## 0. 任务背景

Batch 2.2.b（commit 27971da）已上线，4 字段 UI + 后台调账 4 种新 type 跑通。但**订单业务流**还没接入 4 字段——`consumeBalance` 永远 0。

**Batch 2.2.c 目标**：让 `consumeBalance` 在用户下单付款时自动 +payAmount，退款时自动 -payAmount。

**写入规则**（v43-7-design.md 章节 8.4 已定）：

| 场景 | 触发位置 | consumeBalance 变化 |
|------|---------|-------------------|
| 订单支付 | payOrder / verify-payment | +payAmount |
| 订单退款 | requestRefund | -payAmount |
| 订单取消（pending）| cancelOrder | **不动**（没付款） |

**cancelOrder 不动**——pending 状态订单未付款，`consumeBalance` 本来就是 0，**无需扣减**。

**完整设计文档**：`docs/v43-7-design.md` 章节 8.4 + 8.5

## 2. 改动 1：payOrder 扣 balance 同时 +consumeBalance

**文件**：`src/lib/services/order.service.ts`

**位置**：line 189（事务内 updateMany 段）

**当前代码**（line 186-193）：
```ts
if (order.payAmount > 0) {
  const freshUser = await tx.user.findUnique({ where: { id: order.userId }, select: { balance: true, frozenBalance: true } })
  if (!freshUser) throw new Error('用户不存在')
  const bu = await tx.user.updateMany({ where: { id: order.userId, balance: { gte: order.payAmount } }, data: { balance: { decrement: order.payAmount } } })
  if (bu.count === 0) throw new Error('可用余额不足')
  const nb = freshUser.balance - order.payAmount
  await tx.balanceRecord.create({ data: { userId: order.userId, type: 'payment', amount: -order.payAmount, balance: nb, frozenBalance: freshUser.frozenBalance, sourceType: 'order', sourceId: orderId, description: '订单 ' + order.orderNo + ' 支付' } })
}
```

**改后代码**：
```ts
if (order.payAmount > 0) {
  const freshUser = await tx.user.findUnique({ where: { id: order.userId }, select: { balance: true, frozenBalance: true } })
  if (!freshUser) throw new Error('用户不存在')
  // v43-7 Batch 2.2.c: 扣 balance 同时 +consumeBalance
  const bu = await tx.user.updateMany({
    where: { id: order.userId, balance: { gte: order.payAmount } },
    data: {
      balance: { decrement: order.payAmount },
      consumeBalance: { increment: order.payAmount },  // 新增：消费余额累加
    },
  })
  if (bu.count === 0) throw new Error('可用余额不足')
  const nb = freshUser.balance - order.payAmount
  await tx.balanceRecord.create({ data: { userId: order.userId, type: 'payment', amount: -order.payAmount, balance: nb, frozenBalance: freshUser.frozenBalance, sourceType: 'order', sourceId: orderId, description: '订单 ' + order.orderNo + ' 支付' } })
}
```

**关键点**：
- `data` 对象加 `consumeBalance: { increment: order.payAmount }`
- **`consumeBalance` 不需要 where 条件**——它是累加器，扣 balance 的防透支条件已经保护
- `where` 条件**不动**——只校验 `balance >= payAmount`，`consumeBalance` 不参与
- BalanceRecord 流水**不动**——type=payment 含义不变（只是扣余额），不写 4 字段

## 3. 改动 2：requestRefund 退余额同时 -consumeBalance

**文件**：`src/lib/services/order.service.ts`

**位置**：line 325-348（事务内退余额段）

**当前代码**（line 325-349）：
```ts
if (order.payAmount > 0) {
  const refundUser = await tx.user.findUnique({
    where: { id: order.userId },
    select: { balance: true, frozenBalance: true },
  })
  if (refundUser) {
    await tx.user.update({
      where: { id: order.userId },
      data: { balance: { increment: order.payAmount } },
    })
    const newBalance = refundUser.balance + order.payAmount
    await tx.balanceRecord.create({
      data: {
        userId: order.userId,
        type: 'refund',
        amount: order.payAmount,
        balance: newBalance,
        frozenBalance: refundUser.frozenBalance,
        sourceType: 'order',
        sourceId: orderId,
        description: '订单 ' + order.orderNo + ' 退款',
      },
    })
  }
}
```

**改后代码**：
```ts
if (order.payAmount > 0) {
  const refundUser = await tx.user.findUnique({
    where: { id: order.userId },
    select: { balance: true, frozenBalance: true, consumeBalance: true },  // v43-7 Batch 2.2.c: 多查 consumeBalance
  })
  if (refundUser) {
    await tx.user.update({
      where: { id: order.userId },
      // v43-7 Batch 2.2.c: 退 balance 同时 -consumeBalance
      data: {
        balance: { increment: order.payAmount },
        consumeBalance: { decrement: order.payAmount },
      },
    })
    const newBalance = refundUser.balance + order.payAmount
    await tx.balanceRecord.create({
      data: {
        userId: order.userId,
        type: 'refund',
        amount: order.payAmount,
        balance: newBalance,
        frozenBalance: refundUser.frozenBalance,
        sourceType: 'order',
        sourceId: orderId,
        description: '订单 ' + order.orderNo + ' 退款',
      },
    })
  }
}
```

**关键点**：
- `data` 加 `consumeBalance: { decrement: order.payAmount }`
- **`select` 多查 `consumeBalance: true`**——不是必需，但**显式 select 避免被未来 schema 变化影响**
- **不校验 consumeBalance 是否够扣**——理论上用户付款后 consumeBalance ≥ payAmount（路径 1 接受偶尔越扣的情况，记录为"消费作废"语义）
- BalanceRecord type=refund **不动**——只是退余额流水

**关于 consumeBalance 扣成负数**：
- 路径 1 设计接受（4 字段是"叠加统计"，不强求严格 ≥ 0）
- 如果想防负数：加 where 条件 `consumeBalance: { gte: order.payAmount }` —— 但**本次不做**（避免范围扩散）

## 9. 风险点

1. **consumeBalance 扣成负数**（路径 1 接受）—— 退款时如果用户提现/调账使 consumeBalance < payAmount，会扣成负数
2. **两条入口**（payOrder + verify-payment）—— 必须都改，否则用户从前端 pay 走 verify-payment 时 consumeBalance 不变（这是 v43-4 教训）
3. **历史数据断层**—— Batch 2.2.c 之前所有订单没累加 consumeBalance，**历史订单的 consumeBalance 永远是 0**（接受）
