## 3. 改动 2：TYPE_TABS 加 1 个 "作废" tab

**位置**：line 43-49

**当前代码**：
```ts
const TYPE_TABS: { key: TypeFilter; label: string }[] = [
  { key: 'all', label: '全部' },
  { key: 'payment', label: '消费' },

```

**改后代码**（加 1 个 "作废" tab，**放在奖励之后、提现之前**——因为作废是奖励的反向操作）：
```ts
const TYPE_TABS: { key: TypeFilter; label: string }[] = [
  { key: 'all', label: '全部' },
  { key: 'payment', label: '消费' },
  { key: 'reward', label: '奖励' },
  { key: 'void', label: '作废' },  // v43-7 Batch 2.2.f: 加 1 个 tab
  { key: 'withdraw', label: '提现' },
  { key: 'admin_adjust', label: '管理员调整' },
]
```

**关键点**：
- "作废" 放在"奖励"之后——逻辑上"作废"是"奖励"的反向
- 标签用 "作废"（和 2.2.e 的"已作废收益"呼应）
- 不加 4 字段其他 tab：
  - `earningsPending` 路径 1 永远 0，**不显示**（避免空 tab）
  - `consumeBalance` 已有"消费"tab 覆盖（改动 3 扩展为 payment+refund）
  - `earningsAvailable` 已有"奖励"tab 覆盖

## 4. 改动 3：typeParam switch 扩展

**位置**：line 88（fetchRecords 函数内）

**当前代码**：
```ts
const typeParam = activeTab === 'all' ? '' : activeTab === 'reward' ? 'referral_reward,brand_bonus,dividend_reward,daily_dividend,manual_reward,reward' : activeTab === 'withdraw' ? 'withdraw_freeze,withdraw,unfreeze' : activeTab
```

**改后代码**：
```ts
const typeParam =
  activeTab === 'all' ? '' :
  activeTab === 'reward' ? 'referral_reward,brand_bonus,dividend_reward,daily_dividend,manual_reward,reward' :
  activeTab === 'withdraw' ? 'withdraw_freeze,withdraw,unfreeze' :
  activeTab === 'payment' ? 'payment,refund' :  // v43-7 Batch 2.2.f: 消费 tab 含 refund（consumeBalance 视角）
  activeTab === 'void' ? 'refund_dividend' :    // v43-7 Batch 2.2.f: 作废 tab
  activeTab
```

**关键点**：
- `payment` tab 从单 type 改成 `payment,refund`（消费余额增减完整链路）
- `void` tab 映射到 `refund_dividend`（earningsVoided 主要来源）
- 其他 tab（reward/withdraw/admin_adjust）不动
