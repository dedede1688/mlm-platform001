## 2. 改动 1：package.json 加 test:coverage:check script

**位置**：scripts 段（test:coverage 后面）

**当前代码**：
```json
"test": "vitest run",
"test:watch": "vitest",
"test:coverage": "vitest run --coverage",
"test:services": "npm run build && node test-services.js",
```

**改后代码**：
```json
"test": "vitest run",
"test:watch": "vitest",
"test:coverage": "vitest run --coverage",
"test:coverage:check": "vitest run --coverage --coverage.thresholds.statements=70",
"test:services": "npm run build && node test-services.js",
```

**关键点**：
- 1 行新增
- `test:coverage:check` 是 CI 专用，**只**卡 Statements 70%
- **不**卡 Branches / Functions / Lines（避免立刻 fail CI——Branches 62% 没到 70%）
- 不需要建 vitest.config.ts——CLI 参数够用

## 3. 改动 2：test.yml 加 threshold 步骤

**文件**：`.github/workflows/test.yml`

**位置**：Upload coverage report 步骤之后

**当前代码**：
```yaml
      - name: Lint
        run: pnpm lint
      - name: TypeScript type check
        run: pnpm typecheck
      - name: Run tests
        run: pnpm test
      - name: Run tests with coverage
        run: pnpm test:coverage
      - name: Upload coverage report
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: coverage-report
          path: coverage/
          retention-days: 30
```

**改后代码**（在 Upload 步骤**之后**加 1 步）：
```yaml
      - name: Lint
        run: pnpm lint
      - name: TypeScript type check
        run: pnpm typecheck
      - name: Run tests
        run: pnpm test
      - name: Run tests with coverage
        run: pnpm test:coverage
      - name: Check coverage threshold
        run: pnpm test:coverage:check
      - name: Upload coverage report
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: coverage-report
          path: coverage/
          retention-days: 30
```

**关键点**：
- 加 1 步：`Check coverage threshold`
- **必须**放在 Upload 步骤**之前**（这样如果 threshold fail CI，artifact 仍能上传 for debug）
- `--coverage.thresholds.statements=70` Statements < 70% 即 fail CI
