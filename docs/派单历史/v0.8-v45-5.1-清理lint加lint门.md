### 1.3 `var` → `const`（3 个，order.service.ts:198）

`src/lib/services/order.service.ts:198`：
```typescript
// 原（v43-6 Batch 3 的 var 风格）
var ue=paidOrder.user?.email;var up=paidOrder.user?.phone;var nv={orderNo:paidOrder.orderNo,orderAmount:paidOrder.totalAmount.toFixed(2),payAmount:paidOrder.payAmount.toFixed(2),userName:paidOrder.user?.nickname??paidOrder.user?.phone}

// 改为
const ue=paidOrder.user?.email;const up=paidOrder.user?.phone;const nv={orderNo:paidOrder.orderNo,orderAmount:paidOrder.totalAmount.toFixed(2),payAmount:paidOrder.payAmount.toFixed(2),userName:paidOrder.user?.nickname??paidOrder.user?.phone}
```

**3 个替换**：
- `var ue=` → `const ue=`（line 198:5）
- `var up=` → `const up=`（line 198:34）
- `var nv=` → `const nv=`（line 198:63）

### 1.4 `let` → `const`（1 个）

`src/app/api/admin/referral-tree/[userId]/route.ts:106`：
```typescript
// 原（let 但没重新赋值）
let originalUserId = ...
// 改为
const originalUserId = ...
```

**先确认**：`originalUserId` 在函数内**没重新赋值**（用 grep 验证），再改成 const。

### 1.5 未用参数（1 个）

`src/lib/services/points.service.ts:62`：
```typescript
// 原
static async transferPoints(fromUserId: string, toUserId: string, amount: number, description: string) {
// 改为（参数加 _ 前缀）
static async transferPoints(fromUserId: string, toUserId: string, amount: number, _description: string) {
```

> **参数加 `_` 前缀是安全的**：参数名不影响外部调用方（TypeScript 命名参数不影响 runtime）。
> **确认函数内**：`grep "_description\|description"` 在文件内，**只用到参数名要同步改**——如果函数体内用了 `description` 变量，加 `_` 后会变 unused vars 报错（function 内部引用），需要同步改成 `_description`。
> 已知：transferPoints 是 v45.2 死代码修复（函数体内根本没用 description 参数），所以改 `_description` 不破坏逻辑。

## 2. 关键安全规则（**绝对不能跑 `eslint --fix`**）

### 2.1 为什么 v45.5.1 **禁止 --fix**

v45.5 时小金跑 `eslint --fix` 自动改 16 个业务文件，**差点破坏 JSX 引用**：
- `addSpecValue` 改名 `_addSpecValue` → 如果 JSX 里有 `<button onClick={addSpecValue}>` 引用 → 运行时 `undefined is not a function` ❌
- 删除 unused import `Link` → 如果页面有 `<Link href="...">` 引用 → 编译失败 ❌

### 2.2 v45.5.1 必须**手修 + 验证引用**

**对每个改的函数名/变量名**：
```powershell
# 改之前 grep 找引用
Select-String -Path src/**/*.tsx, src/**/*.ts -Pattern "addSpecValue"
```

**有引用 → 保留函数，加 `_` 前缀**（lint 允许）
**无引用 → 才考虑删除**（但建议还是加 `_` 前缀，更稳）

**对每个改的 import 删除**：
```powershell
# 改之前确认无引用
Select-String -Path src/**/*.tsx -Pattern "<Link\b|\bLink\."
```

**有引用 → 不删，改成 `import Link from 'next/link'`**
**无引用 → 才删除**

## 3. 改动 2：CI workflow 加回 Lint 步骤

### 3.1 `.github/workflows/test.yml`

**当前**（v45.5 加了 typecheck，去掉了 lint）：
```yaml
      - run: pnpm install --frozen-lockfile
      - name: TypeScript type check
        run: pnpm typecheck
      - name: Run tests
        run: pnpm test
```

**改为**（加回 Lint 在 typecheck 之前）：
```yaml
      - run: pnpm install --frozen-lockfile
      - name: Lint
        run: pnpm lint
      - name: TypeScript type check
        run: pnpm typecheck
      - name: Run tests
        run: pnpm test
```

**步骤顺序**：Lint → Typecheck → Test（lint 快，先挡明显错；typecheck 中等；test 最后最慢）。
