## 0. 任务背景

v45.4 7a33358 CI 跑通（47/47 ✅），但 CI 只跑 `pnpm test`，**没有 lint 和 typecheck 防线**。

v45.5 在 CI 加 lint + typecheck 强制门：
- 业务代码有 TS 类型错 → CI 阻止 push
- 业务代码有 ESLint 错 → CI 阻止 push

**为什么是"强制门"**：
- 现在 `pnpm test` 过了不代表代码健康（vitest 只跑测试文件，业务代码类型错测试不一定覆盖到）
- 比如 v8 教训里 v43-4-修复-2 P0 bug：verify-payment 重复调 payOrder，**类型完全正确**但业务逻辑错
- lint + typecheck 是底线门，挡住的是"明显错误"，复杂业务逻辑靠测试和 review

## 3. 改动 2：`.github/workflows/test.yml` 加 lint + typecheck 步骤

**改前**（约第 1-20 行）：
```yaml
name: Tests
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
        with:
          version: 9
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: pnpm
      - run: pnpm install --frozen-lockfile
      - run: pnpm test
```

**改后**：
```yaml
name: Tests
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
        with:
          version: 9
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: pnpm
      - run: pnpm install --frozen-lockfile
      - name: Lint
        run: pnpm lint
      - name: TypeScript type check
        run: pnpm typecheck
      - name: Run tests
        run: pnpm test
```

**顺序**：lint → typecheck → test（lint 先快、typecheck 中等、test 最后）。

**name**: 加 `name:` 字段让 CI 日志可读（GitHub Actions 步骤名）。

## 10. 真实风险参考

**风险 1：`pnpm lint` 用 `next lint`，Next.js 15 已 deprecated**
- 不影响跑（还能用），只是有 runtime warning
- v45.5 不动这个，避免引入风险
- 后续 v45.5.1 改成 `eslint .` 直接调用（如需）

**风险 2：tsconfig `strict: true` 可能报测试文件类型错**
- 测试文件用了 `mockPrisma: any`（显式 any，不报 noImplicitAny）
- 但 mock chain 类型推断可能有问题（如 `vi.mocked(...)` 返回类型）
- 让小金先跑看，根据真实报错决定怎么修

**风险 3：build 预渲染错误（与 v45.5 无关）**
- `npx next build` 可能报预渲染错误（如某个页面 client-only 组件）
- 这是历史问题，v45.5 之前就存在
- typecheck 用 `tsc --noEmit` 不走 next build，不会暴露这个
- lint 用 `next lint` 也不走 prerender
- 所以 CI 跑 lint + typecheck + test **应该不会受 build 预渲染错误影响**
