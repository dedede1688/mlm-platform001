## 7. 重要认知（Mavis 写给胡子哥参考）

**为什么 v45.6 实际是"流程验证"不是"写 CI"**：

| 任务 | 实现位置 |
|------|----------|
| 跑 lint / typecheck / test | **GitHub Actions**（v45.3-v45.5.1 已做）|
| push main → Production | **Vercel GitHub App**（已自动）|
| push PR → Preview URL | **Vercel GitHub App**（已自动，待验证）|
| 部署通知 / 自定义 build | `vercel.json` + vercel CLI |
| Vercel 项目配置（env / domain） | Vercel Dashboard |

**Vercel vs GitHub Actions 边界**：
- Vercel 管"部署 + 域名 + env"
- GitHub Actions 管"测试 + lint + typecheck + 自定义 CI"

v45.6 跨在 Vercel 侧，不是 CI 侧。所以**不需要改 `.github/workflows/test.yml`**。
