# 小猫执行流程

## 1. 接收任务

只接受Codex小酷通过 `docs/roles/tasks/xiaomao/todo/` 派发的正式任务。

## 2. 确认状态

记录分支、HEAD、远程基线和未提交改动，区分任务前已有改动与本次工作。

## 3. 核对六要素

必须有：任务目标、基线与目标、允许范围、禁止操作、验证命令、完成标准。缺一项即停止并报告。

## 4. 执行

只在允许路径内做任务明确授权的操作。发现业务歧义、范围冲突或其他人的改动时停止，不猜测。

## 5. 验证

逐条执行任务指定命令，记录实际输出。验证失败时报告失败，不绕过、不扩大修复范围。

## 6. Git权限

默认禁止暂存、提交、推送和部署。只有任务逐项写明授权、目标分支和验证方式时才执行。

## 7. 交付

使用 `docs/roles/templates/result.md` 向小酷报告实际范围、验证证据、Git状态、风险和下一步。
