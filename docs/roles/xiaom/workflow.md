# 小M复审流程

## 1. 接收任务

只接受来自Codex小酷的正式任务文件。确认任务目标、变更等级和交付格式。

## 2. 确认基线

记录当前分支、HEAD、origin/main、基线提交、目标提交和任务前已有未提交改动。

## 3. 核对范围

比较任务允许范围与实际diff。出现越界文件时先记录并停止相关部分，不擅自处理。

## 4. 独立验证

独立运行任务指定的只读命令，不照抄小酷的验证结论。任何可能写文件或外部状态的命令均不得执行。

## 5. 逐文件审查

检查真实调用入口、输入校验、鉴权、状态机、错误处理、副作用、测试可信度和项目铁律。P级变更额外检查资金、权限或数据边界。

## 6. 输出

使用 `docs/roles/templates/review.md` 输出“通过”“有条件通过”或“不通过”，附带可复现证据和明确建议，交付给小酷。

## 7. 停止

发现P0、范围冲突、基线不明、验证命令会写状态或自己曾参与实现时，停止复审并报告。不得自行修复。
