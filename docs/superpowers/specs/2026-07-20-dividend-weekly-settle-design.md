## 1. 背景与目标

PRD §2.4.3 规定分红"按天快照等级，**每周统一发放**"。当前实现为**每日结算**：
- `DividendService.settleDailyDividends()` 每日调用；
- `lib/utils/cron.ts` `runDailyTasks()` 调它；
- Vercel cron `0 0 * * *` 触发 `/api/cron/daily-tasks`。

**目标**：改为"每日仍做等级快照，每周仅在指定周几统一发放分红"，与 PRD 对齐，且幂等、可回滚、不重复发放。

## 2. 关键设计决策（待胡子老师确认）

| 决策点 | 建议方案 | 需确认 |
|---|---|---|
| 周结日 | **每周日 00:00（北京时间）** 统一发放上一周分红 | 周几？几点？ |
| 每日是否仍写快照 | **是**，每日只记录 `dividend` 明细 + 等级快照，**不入账**；周结时把"未发放"明细统一入账 | 是否每日记账？ |
| 幂等 / 防重复 | 引入"已结算标记"（如 `dividend.settled` 或结算批次表），周结只发"未结算"明细；或按周期唯一键幂等 | 选哪种幂等 |
| 余额入账字段 | 与现有日结一致：写 `earningsAvailable`（可提现收益），**不**改 `balance` | 是否沿用 |

> 注：当前日结实现（v2 五级独立池）已把分红写入 `earningsAvailable` 并记 `BalanceRecord type='daily_dividend'`。周结需把"发放动作"从每日挪到每周，明细仍每日生成。

## 4. 幂等与回滚

- **幂等**：周结只处理"未结算"明细，且用事务保证"入账 + 标记已结算"原子性；重复触发不会重复发放。
- **回滚**：保留 `dividend` 明细不变，仅调整"何时入账"，可随时切回日结（改回 cron/调用即可）。
- **边界**：某周无订单 → 分红池为 0，不报错（与现有边界处理一致）。

## 5. 测试计划（TDD）

- 周一~周六触发：只快照、不入账。
- 周日触发：把当周"未结算"明细统一入账一次，且只发一次。
- 重复周日触发：第二次 0 新发放（幂等）。
- 跨周快照数据完整、金额正确。
