### 🟢 P2-1：catch 中用错误消息字符串匹配状态码

**位置**：`src/app/api/admin/users/[id]/balance/route.ts` line 288-296

```typescript

  const isBusinessError = message === '可用收益不足'  // ← 字符串精确匹配
  return NextResponse.json(
    { success: false, message },
    { status: isBusinessError ? 400 : 500 }
  )

```

**问题**：
- 这是任务文件第 128 行明确允许的"最小改动"方案（"允许采用最小改动：在路由外层 `catch` 中根据错误消息映射状态码"）

**真实风险**：低（中文错误信息文本不会被随意改），但代码异味

**改进建议**（不阻塞本轮）：
1. 创建 `BusinessError` 自定义 Error 类，catch 中用 `instanceof` 判断
2. 或在事务返回结果中带 status code（`{ code: 'INSUFFICIENT_EARNINGS', status: 400 }`）
3. 通用调账分支（line 174）的 `${mapping.label}不足` 也用同样模式——但任务文件第 312 行 P0 判定"未知异常错误地返回 400"意味着不能让所有错误都返回 400
