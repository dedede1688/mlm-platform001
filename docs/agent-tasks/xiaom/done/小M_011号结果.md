## 5. `users/route.ts` 字段排除行为保留

文件 `src/app/api/admin/users/route.ts` line 99-113：

```typescript
// 格式化返回数据
const data = users.map(u => {
  const orderCount = u.orders?.length || 0
  const totalOrderAmount = u.orders?.reduce((sum, o) => sum + (o.payAmount || 0), 0) || 0
  const { orders: _orders, _count, referrerId: _referrerId, ...rest } = u
  return {
    ...rest,
    referrer: u.referrer
      ? { id: u.referrer.id, nickname: u.referrer.nickname, phone: u.referrer.phone }
      : null,
    directReferralCount: _count?.referrals || 0,
    orderCount,
    totalOrderAmount,
  }
})
```

**关键点**：
- 解构别名后，`orders` / `referrerId` 仍被抽出，`...rest` 仍然**不包含这两个字段**（与原写法语义完全一致）
- `_count` 同样被抽出，不通过 `...rest` 泄露（直接通过 `directReferralCount: _count?.referrals || 0` 显式取）
- `orderCount` / `totalOrderAmount` 从 `u.orders` 显式取，**不通过 `...rest`**，所以即使 `orders` 数组本身被排除，下游依然能拿到聚合值
- 返回给前端的字段集合与原版**完全一致**：`...rest`（不含 `orders` / `_count` / `referrerId`）+ `referrer` + `directReferralCount` + `orderCount` + `totalOrderAmount`

✅ **接口字段排除行为完整保留，未造成 `orders` 数组或 `referrerId` 泄露**。
