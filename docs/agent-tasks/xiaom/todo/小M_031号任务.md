## 五、P1-2 按钮定位核验

必须确认：

1. 先通过 `matchAll(/<button...<\/button>/g)` 或等价方式枚举完整按钮；
2. 再按“重新确认支付”筛选；
3. 断言筛选结果恰好为 1；
4. disabled 只在该目标按钮字符串上检查；
5. 完整场景也只在该目标按钮上断言非 disabled；
6. 删除了可能跨越多个 `</button>` 的旧正则；
7. 没有 `if (match)` 或宽松 OR 假通过。

重点尝试证明旧正则风险已消失：即使页面未来出现其他 disabled 按钮，测试也不会错误地把它与“重新确认支付”拼接匹配。
