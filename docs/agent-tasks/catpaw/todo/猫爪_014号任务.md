在打开新结算弹窗前，根据 `pendingOrder?.cartItemId` 判断是否是同一商品。推荐逻辑：

## 八、必须自检的业务场景

通过源码和测试确认以下状态流：

1. 商品 A 余额不足，生成待支付订单 A，并显示缺口 A。
2. 关闭结算弹窗时，待支付订单 A 和缺口 A 仍保留，但收益转余额弹窗关闭。
3. 重新打开商品 A 时，继续显示缺口 A，并复用订单 A。
4. 打开商品 B 时，清空缺口 A，不显示 A 的余额提示。
5. 商品 B 首次确认时可创建订单 B；不会误用订单 A。
6. 收益转入成功后只刷新资金和更新提示，不自动调用支付接口。
