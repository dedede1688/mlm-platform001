# 猫爪_003号任务：后台会员详情补收益账户 + 可用收益增加入口

## 一、任务背景

当前后台「会员管理」里的「会员详情」弹窗只显示：

- 余额
- 冻结余额
- 总积分
- 可用/锁定积分

但是没有展示用户的收益账户字段。胡子哥在测试「收益转余额」时，需要给测试用户手动增加一笔「可用收益」，再从用户端验证「收益转余额」和「余额不足时引导转入余额」流程。

后端调账接口已经支持 `earnings_add`（增加可用收益），所以本任务只做后台前端页面补入口。


## 六、具体实现要求

### 1. 补充前端用户类型字段

检查 `UserRow` 或页面里使用的用户类型，确保包含以下字段：

```typescript
earningsPending?: number
earningsAvailable?: number
earningsFrozen?: number
earningsVoided?: number
```

如果已有其中部分字段，只补缺失字段。

展示时必须用 `?? 0` 兜底，避免老数据为空导致页面报错。

### 2. 会员详情弹窗增加「收益账户」区域

在「会员详情」弹窗中新增一个清晰的区域，标题建议：

```text
收益账户
```

至少展示 4 个字段：

| 字段 | 页面中文 |
|---|---|
| `earningsAvailable` | 可用收益 |
| `earningsFrozen` | 冻结收益 |
| `earningsPending` | 待结算收益 |
| `earningsVoided` | 作废收益 |

金额格式沿用当前页面已有的金额格式，例如：

```text
¥1000.00
```

### 3. 调账类型增加「可用收益」

当前调账类型大概率只有：

- 余额
- 冻结余额

请增加一个类型：

```text
可用收益
```

提交给后端时，字段值必须是：

```typescript
earnings_add
```

后端已经支持 `earnings_add`，不要改后端。

### 4. 限制本包只允许增加可用收益

当调账类型是 `earnings_add` 时：

- 金额必须大于 0。
- 如果金额小于等于 0，前端直接提示：

```text
本次只允许增加可用收益
```

不要在本包做「减少收益」或「作废收益」。

### 5. 当前值、确认弹窗、成功提示都要识别收益类型

如果页面里有确认弹窗或二次确认，请补齐中文展示：

| 类型 | 中文展示 |
|---|---|
| `balance` | 余额 |
| `frozenBalance` | 冻结余额 |
| `earnings_add` | 可用收益 |

确认弹窗里当前值也要正确：

- `balance` 用用户余额。
- `frozenBalance` 用冻结余额。
- `earnings_add` 用可用收益。

### 6. 成功后刷新

调账成功后，沿用页面现有刷新逻辑：

- 刷新会员列表。
- 如果当前打开了会员详情，也要能看到最新可用收益。

如果已有刷新逻辑，不要重复造新逻辑。
