## 三、已确认问题

### P0：资金成功后通知异常可能让接口返回失败

当前 `notifyEarningsVoid`（收益作废通知）中的用户查询：

```typescript
const user = await prisma.user.findUnique(...)
```

位于异常保护之外。

可能发生：

```text
资金事务已经成功提交
→ 通知查询用户失败
→ 路由返回 500
→ 管理员误以为失败并重试
→ 同一笔收益被重复作废
```

### P1：非有限数字没有完整拦截

当前使用 `isNaN`，不能拒绝 `Infinity`（无穷大）和 `-Infinity`（负无穷大）。

### P1：并发时流水和日志的旧值可能过期

当前先读取 `before`，再执行条件更新。并发等待后，`before` 可能不是本次更新真正发生前的值，导致资金流水说明和操作日志快照不准确。

### P1：没有资金自动化测试

收益作废属于资金变动，不能只依赖类型检查和构建。

### 部署前置项：生产通知模板尚未同步

`earnings_voided` 模板目前只写入 `prisma/seed.ts`。本任务不连接生产数据库，不运行全量 seed（种子数据命令）。模板同步留到代码复审通过后的单独部署步骤。

### 任务 2：修复通知方法的完整异常保护

修改：

```text
src/lib/services/order-notification.service.ts
```

`notifyEarningsVoid` 的异常保护必须覆盖整个方法，包括：

- `prisma.user.findUnique`
- `notificationBatch.create`
- `sendInApp`

不要只包内部通知创建代码。

推荐结构：

```typescript
static async notifyEarningsVoid(params: ...) {
  try {
    const user = await prisma.user.findUnique(...)
    if (!user) return

    const batch = await prisma.notificationBatch.create(...)
    await sendInApp(...)
  } catch (error) {
    console.error(...)
    logger.error(...)
  }
}
```

删除没有必要的立即执行异步函数包装，不要使用未等待的异步调用。

### 任务 3：在路由层增加第二道通知失败隔离

修改：

```text
src/app/api/admin/users/[id]/balance/route.ts
```

即使通知方法未来再次出现未捕获异常，路由也不能把已经成功的资金操作返回成失败。

要求：

```typescript
await OrderNotificationService.notifyEarningsVoid(...).catch((error) => {
  console.error(...)
  logger.error(...)
})
```

如路由尚未导入 `logger`（日志工具），可以从项目现有日志模块导入。

必须保留 `await`，不能改成不等待的调用。

其他调账通知逻辑不要扩大修改范围。

### 任务 4：使用有限数字校验

将金额校验改为拒绝所有非有限数字：

```typescript
if (typeof amount !== 'number' || !Number.isFinite(amount) || amount === 0) {
  // 返回 400
}
```

确认以下输入均返回 400，并且不进入资金事务：

- 字符串 `"40"`
- `NaN`
- `Infinity`
- `-Infinity`
- `0`

`earnings_void` 还必须拒绝负数。

### 任务 5：修复并发下的真实快照

在 `updateMany` 成功后，应先读取数据库更新后的真实用户值，再构造流水和日志。

本次操作前的真实收益值可以由更新后的值反推：

```typescript
const actualOldValue = {
  earningsAvailable: updated.earningsAvailable + amount,
  earningsVoided: updated.earningsVoided - amount,
}
```

要求：

- `BalanceRecord`（资金流水）说明使用本次操作真实前值和真实后值。
- `OperationLog.oldValue` 使用真实前值。
- `OperationLog.newValue` 使用数据库读取的真实后值。
- 通知使用数据库读取的真实后值。
- 不使用 `before.earningsAvailable - amount` 伪造更新后快照。
- `balance` 和 `frozenBalance` 流水快照使用更新后查询到的真实值。

数据库行更新仍必须保持：

```typescript
updateMany({
  where: { id, earningsAvailable: { gte: amount } },
  data: {
    earningsAvailable: { decrement: amount },
    earningsVoided: { increment: amount },
  },
})
```

不得改成先读后普通 `update`，不得使用原始 SQL。
