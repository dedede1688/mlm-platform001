## 二、重要技术判断

### 2.1 不要收紧整个会员详情接口权限

`GET /api/admin/users/[id]` 仍然必须允许：

```typescript
verifyPermission(request, ['support_admin', 'super_admin'])
```

原因：客服管理员仍需查看会员详情。只有“重置支付密码”接口必须限制为 `super_admin`（超级管理员）。

禁止把会员详情接口改为只允许 `super_admin`。

### 2.2 敏感字段处理规则

会员详情接口必须使用 Prisma `select`（字段白名单）读取必要字段，禁止继续使用：

```typescript
include: { ... }
return { ...user }
```

允许在 `select` 中读取 `paymentPasswordHash`，但只能用于计算：

```typescript
hasPaymentPassword: Boolean(paymentPasswordHash)
```

最终响应中绝对不能出现：

- `passwordHash`
- `paymentPasswordHash`

不要查询 `passwordHash`。不要用“先查完整用户、再 delete”的方式代替字段白名单。

```typescript

  success: true,
  data: {
    ...safeUser,
    hasPaymentPassword: Boolean(paymentPasswordHash),
    orderCount: orderStats._count,
    totalOrderAmount: orderStats._sum.payAmount || 0,
  },
  message: '获取会员详情成功',
})
```
