### 4.1 后端退款防重复

退款 POST 接口必须把进行中查询改为：

```typescript
where: {
  orderId,
  status: { in: ['pending', 'approved'] },
}
```

统一错误：

```text
该订单已有进行中的退款申请
```

必须验证：

- pending 拒绝；
- approved 拒绝；
- rejected 允许重新申请；
- 拒绝时不 create、不写日志、不发通知；
- 原有鉴权、归属、状态、原因和成功通知保持。

### 4.2 售后服务组件

新建真实生产组件 `OrderAfterSalesCard.tsx`，集中处理：

- 无申请：小号描边“申请退款”；
- pending：黄色“审核中”，无按钮；
- approved：蓝色“退款处理中”，无按钮；
- completed：绿色“退款已完成”，无按钮；
- rejected：红色状态，唯一小号“重新申请”；
- 未知状态：中性文字，无申请按钮；
- 原生 `<details>` / `<summary>` 展开退款原因、说明、管理员备注、凭证和时间；
- 桌面左右排列，移动端上下排列；
- 不使用嵌套卡片；
- 不出现 `w-full py-3 text-base bg-orange-600` 大按钮样式。

### 4.3 订单详情页接入

金额明细之后只渲染一次 `OrderAfterSalesCard`。

删除：

- 旧退款状态卡片；
- 底部满宽橙色“申请退款”；
- 重复的“退款申请审核中”；
- rejected 状态的重复入口。

保留退款申请成功提示、待支付按钮和收益转余额弹窗。

### 4.4 测试可信度

组件测试必须真实导入生产 `OrderAfterSalesCard`，使用 `renderToStaticMarkup` + `React.createElement` 渲染。

禁止：

- mock 生产组件；
- 复制简化组件；
- `if (match)` 静默跳过；
- 宽松 `A || B`；
- 固定字符窗口；
- 可跨 `</button>` 的正则。

按钮检查必须先枚举完整按钮，再按文字筛选并断言唯一数量。
