## 四、第一阶段：生产通知模板单条同步

### 1. 禁止全量 seed

### 2. 创建一次性脚本

临时文件路径：

```text
scripts/_tmp-upsert-earnings-voided-template.ts
```

使用 `apply_patch`（补丁工具）创建，不要使用 `cat`、重定向或 Python 写文件。

脚本只允许操作 `notificationTemplate`，内容必须等于 `prisma/seed.ts` 当前定义：

```typescript
import { PrismaClient } from '@prisma/client'

const prisma = new PrismaClient()

const template = {
  id: 'seed-tpl-earnings_voided-in_app',
  type: 'earnings_voided',
  channel: 'in_app',
  subject: '收益作废通知',
  content: '【敏维科技】{{userName}} 您好，您的可用收益 ¥{{amount}} 已被后台作废。剩余可用收益：¥{{earningsAvailable}}，累计作废收益：¥{{earningsVoided}}。如有疑问请联系客服。',
  enabled: true,
}

async function main() {
  const existing = await prisma.notificationTemplate.findUnique({
    where: {
      type_channel: {
        type: template.type,
        channel: template.channel,
      },
    },
    select: { id: true },
  })

  await prisma.notificationTemplate.upsert({
    where: {
      type_channel: {
        type: template.type,
        channel: template.channel,
      },
    },
    update: {
      subject: template.subject,
      content: template.content,
      enabled: template.enabled,
    },
    create: template,
  })

  const verified = await prisma.notificationTemplate.findUnique({
    where: {
      type_channel: {
        type: template.type,
        channel: template.channel,
      },
    },
    select: {
      id: true,
      type: true,
      channel: true,
      subject: true,
      content: true,
      enabled: true,
    },
  })

  if (
    !verified ||
    verified.type !== template.type ||
    verified.channel !== template.channel ||
    verified.subject !== template.subject ||
    verified.content !== template.content ||
    verified.enabled !== true
  ) {
    throw new Error('earnings_voided 模板同步后验证失败')
  }

  console.log(JSON.stringify({
    action: existing ? 'updated' : 'created',
    template: verified,
  }, null, 2))
}

main()
  .catch((error) => {
    console.error('模板同步失败：', error instanceof Error ? error.message : String(error))
    process.exitCode = 1
  })
  .finally(async () => {
    await prisma.$disconnect()
  })
```

如果 `.env.local` 不包含数据库连接变量，再使用：

如果遇到连接池临时错误，可以重试一次；连续失败时停止，不要改用全量 seed，不要继续提交部署。

## 七、第四阶段：提交和推送

提交信息：

```text
feat(admin): add earnings void adjustment
```

如果 PowerShell 对引号或括号解析异常，使用临时提交信息文件配合 `git commit -F`，成功后立即删除临时文件，不能让临时文件进入提交。

提交成功后记录完整提交编号，然后运行：

```powershell
git push origin main
git log origin/main --oneline -1
```

铁律 1：

- `git log origin/main --oneline -1` 必须显示本次新提交编号。
- 本地 `HEAD` 必须等于 `origin/main`。
- 不一致时视为推送失败，不能报告成功。
