# 猫爪_022号结果：支付密码重置手机号后四位前端校验修复

## 1. 最终结论

**完成。** 支付密码重置时，错误手机号后四位现在会被前端即时拦截，不再允许打开二次确认弹窗或调用重置接口。

## 2. 根因确认

前端 `handleResetPaymentPassword` 只验证了 `/^\d{4}$/` 格式，没有比较输入值是否等于 `detailUser.phone.slice(-4)`。同时 `doResetPaymentPassword` 没有二次校验，接口失败时只读 `data.message` 而非 `data.error`。

- 失败原因：前端代码缺少 `detailUser.phone.slice(-4)` 比较、缺少红色即时提示、按钮 disabled 不含匹配条件、错误分支未读 `data.error`

## 5. handleResetPaymentPassword 如何校验真实后四位

在函数外部定义了 3 个计算变量（line 206-208）：

```typescript
const actualPhoneSuffix = detailUser?.phone ? detailUser.phone.slice(-4) : ''
const normalizedSuffix = payPwdResetSuffix.trim()
const suffixMatches = /^\d{4}$/.test(normalizedSuffix) && normalizedSuffix === actualPhoneSuffix
```

`handleResetPaymentPassword` 在通过格式校验后新增：

```typescript
if (normalizedSuffix !== actualPhoneSuffix) { showMessage('error', '手机号后 4 位不匹配，请核对后重试'); return }
```

不匹配时在 `setShowPayPwdConfirm(true)` 之前 return，不会打开确认弹窗。

## 6. doResetPaymentPassword 如何阻止错误值调用接口

在 `setSavingPayPwdReset(true)` 和 `fetch()` 之前新增：

```typescript
if (normalizedSuffix !== actualPhoneSuffix) {
  showMessage('error', '手机号后 4 位不匹配，请核对后重试')
  setShowPayPwdConfirm(false)
  return
}
```

不匹配时：不进入 saving 状态、不调用接口、关闭确认弹窗、立即 return。

## 7. 输入框即时错误提示如何实现

输入框下方条件渲染（line 1155-1156）：

```tsx
{normalizedSuffix.length === 4 && normalizedSuffix !== actualPhoneSuffix && (
  <p className="mt-1 text-xs text-red-500">手机号后 4 位不匹配，请核对后重试</p>
)}
```

输入框边框也随状态变色（line 1154）：

```tsx
className={`... ${normalizedSuffix.length === 4 && normalizedSuffix !== actualPhoneSuffix ? 'border-red-400' : 'border-gray-300'}`}
```

输入正确或未满 4 位时不显示提示。

## 8. 按钮禁用条件如何实现

按钮 disabled 使用 `!suffixMatches`（line 1160）：

```tsx
disabled={savingPayPwdReset || !payPwdResetReason.trim() || payPwdResetReason.trim().length < 5 || !suffixMatches}
```

`suffixMatches` 定义为 `/^\d{4}$/.test(normalizedSuffix) && normalizedSuffix === actualPhoneSuffix`，同时满足 4 位数字格式和与真实后四位匹配。

## 9. 后端 data.error 如何正确展示

`doResetPaymentPassword` 错误分支改为（line 236）：

```typescript
showMessage('error', data.error || data.message || '重置失败')
```

优先读取 `data.error`，确保后端返回"手机号后 4 位不匹配"时前端能正确显示。
