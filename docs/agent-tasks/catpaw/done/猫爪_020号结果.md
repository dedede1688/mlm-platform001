## 六、Prisma `select` 投影 mock 如何实现

### 辅助函数

```typescript
// v020: 模拟 Prisma select 字段白名单投影（仅顶层标量+关系字段）
function applyTopLevelSelect(record: Record<string, unknown>, select: Record<string, unknown>) {
  return Object.fromEntries(
    Object.keys(select).map(key => [key, record[key]])
  )
}
```

### Mock 辅助函数

```typescript
// v020: 使用 select 投影的 mock 辅助函数
function mockFindUniqueWithSelect(record: Record<string, unknown>) {
  ;(prisma.user.findUnique as any).mockImplementation((args: any) => {
    return Promise.resolve(applyTopLevelSelect(record, args.select))
  })
}
```

### 工作原理

1. 路由调用 `prisma.user.findUnique({ where: {...}, select: {...} })` 时，mock 拦截到 `args.select`
2. `applyTopLevelSelect` 遍历 `select` 对象的 key，从完整数据库记录 `FULL_USER` 中只提取这些 key 对应的值
3. 因为路由的 `select` 不包含 `passwordHash`，所以投影后的返回对象也不包含 `passwordHash`
4. 因为路由的 `select` 包含 `paymentPasswordHash`，所以投影后的返回对象包含该字段（供路由计算布尔状态）
5. 嵌套关系字段（`referrer`、`parent`、`referrals`、`children`）也通过 `record[key]` 直接取测试记录中的值

### 替换范围

所有原来使用 `;(prisma.user.findUnique as any).mockResolvedValue(FULL_USER)` 的测试用例均已替换为 `mockFindUniqueWithSelect(FULL_USER)`。测试 6/7（自定义 paymentPasswordHash 值）和测试 14（status=deleted）也同步替换。
