### 6.1 测试假通过专项检查

当前组件测试中可能存在以下写法：

```typescript
if (closeBtnMatch) {
  expect(...)
}
```

这种写法在匹配失败时会跳过断言并假通过。必须检查：

1. 右上角关闭按钮测试是否先强制 `expect(closeBtnMatch).not.toBeNull()`，再检查 `closeDetailModal`。
2. 底部关闭按钮测试是否先强制 `expect(closeBtnArea).not.toBeNull()`，再检查 `closeDetailModal`。
3. 所有关键函数体提取失败时是否明确失败，而不是静默跳过。
4. 正则提取是否只覆盖目标函数体，避免跨函数误匹配。

如果业务实现正确但测试会假通过，列为 P1，要求补测试后再发布。
