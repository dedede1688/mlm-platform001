## 三、背景说明

胡子哥希望后台「会员详情」弹窗不要继续长滚动，而是像浏览器网页一样，用标签页切换内容。

本任务目标：

1. 会员详情弹窗改为标签页结构。
2. 新增或整理出「资金账户」标签。
3. 「资金账户」里集中展示余额账户、收益账户、资金调整操作。
4. 保留已有 `earnings_add`（增加可用收益）操作。

## 四、重点检查项

### 1. 修改范围

确认业务代码是否只修改：

```text
src/app/admin/users/page.tsx
```

不应修改：

- 后端接口
- 数据库结构
- `prisma/schema.prisma`
- `AGENTS.md`
- 支付逻辑
- 订单逻辑
- 提现逻辑
- 收益转余额后端逻辑

### 2. 标签页结构

检查会员详情弹窗是否已经从长滚动堆叠，改为标签页切换。

至少应包含：

- 基本资料
- 资金账户
- 经营统计
- 推荐关系
- 直推列表

打开会员详情时，默认应显示「基本资料」。

### 3. 资金账户标签

检查「资金账户」标签中是否包含：

#### 余额账户

- 余额
- 冻结余额
- 消费余额（如果页面已有该字段）

#### 收益账户

| 字段 | 中文展示 |
|---|---|
| `earningsAvailable` | 可用收益 |
| `earningsFrozen` | 冻结收益 |
| `earningsPending` | 待结算收益 |
| `earningsVoided` | 作废收益 |

金额字段应有 `?? 0` 兜底。

#### 资金调整操作

资金调整应在「资金账户」标签中可以看到。

### 4. `earnings_add` 检查

确认「可用收益」调账提交类型仍然是：

```text
earnings_add
```

不要变成其他值。

### 5. 0 或负数拦截

当调账类型为 `earnings_add` 时：

- 金额必须大于 0。
- 0 或负数必须前端拦截。
- 提示文案应类似：

```text
本次只允许增加可用收益
```

### 6. 禁止项检查

确认没有新增：

- 收益扣减
- 作废收益操作
- 余额转回收益

确认没有修改：

- 后端接口
- 数据库结构
- 支付逻辑
- 提现逻辑
- 订单逻辑

### 7. 验证命令结果

请检查或自行只读运行：

```powershell
npx tsc --noEmit -p tsconfig.typecheck.json
npx next build
```

如果运行命令，只允许验证，不允许修改代码。
