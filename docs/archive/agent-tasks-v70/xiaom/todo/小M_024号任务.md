## 五、最高优先级复审项

### P0 候选 1：待支付锁定后是否仍校验隐藏的地址字段

逐行检查 `CheckoutDialog.tsx`：

1. `hasPendingOrder=true` 时，是否无条件隐藏可编辑地址表单；不能使用 `hasPendingOrder && lockedShipping ? ... : 可编辑表单` 让快照为空时回到可编辑表单。
2. `handleConfirm` 在待支付状态下是否使用 `lockedShipping`，并跳过隐藏的 `recipientName`、`recipientPhone`、省市区和详细地址校验。
3. 模拟以下真实路径：
   - 用户没有地址簿默认地址；
   - 首次填写新地址并创建订单；
   - 余额不足；
   - 关闭结算弹窗，等待关闭清理定时器执行；
   - 再次打开同一待支付订单；
   - 只输入支付密码重新确认。
4. 判断该路径是否会错误提示“请输入收件人姓名”或“请选择完整地址”，导致无法继续支付。
5. 商品详情充值返回后使用 `pendingOrderShipping` 时，也要检查隐藏字段校验是否仍会阻断。

如果真实路径会被阻断，列为 P0，不得判定通过。

### P0/P1 候选 2：订单详情白名单是否破坏既有页面

检查现有真实消费者：

```text
src/app/dashboard/orders/[id]/page.tsx
src/app/payment/order/[orderId]/page.tsx
src/app/api/orders/[id]/cancel/route.ts
src/app/api/orders/[id]/route.ts 的 PUT / DELETE
```

重点确认：

- `paymentVerified（支付验证状态）` 是否仍由服务层查询并由 GET 公开响应返回；
- 订单详情页实际使用的每个字段是否仍存在；
- PUT、DELETE、cancel 路由所需 `userId` 和 `status` 是否仍存在；
- 敏感字段 `user`、`userId`、`passwordHash`、`paymentPasswordHash` 不得出现在 GET 对外响应。

既有页面依赖字段被删，按影响判为 P0 或 P1。

### P1 候选 3：恢复资金读取失败是否伪造零缺口

检查 `restorePendingPayment`：

- `/api/users/me` 失败或返回非成功时，是否把 `shortage` 直接设为 `0`；
- 临时资金读取失败应进入 `validation_error`，保留会话并禁止创建新订单；
- 不得显示“已恢复”或让用户误以为无需补余额。

### P1 候选 4：会话工具数值校验

检查 `calculatePendingShortage(payAmount, balance)`：

- `NaN`、`Infinity`、`-Infinity`、负数是否有明确处理；
- 实现是否符合设计计划中的有限数字校验；
- 测试是否真实覆盖这些输入，而不是只覆盖正常金额。

### P1 候选 5：测试是否存在假通过

重点检查：

- `product-pending-payment-flow.test.ts` 是否存在 `if (match) { expect(...) }`；
- `checkout-pending-order-lock.test.ts` 是否只搜索任意 `hasPendingOrder ?`，但没有证明地址表单真的被锁定；
- 是否用全文件 `toContain` 代替真实函数体或分支提取；
- “重新验证”测试是否真实断言 `restorePendingPayment(true)`；
- “不自动支付”测试在函数提取失败时是否仍会通过。

任何能静默跳过的关键断言至少列为 P1 测试缺口。

## 六、完整业务链路推演

必须独立推演并写出结论：

1. 首次立即购买 → 创建一个订单 → 写入内存和会话 → 验证支付。
2. 余额不足、有收益 → 打开收益转余额 → 转入成功 → 不自动支付。
3. 余额不足、无收益 → 保留同一订单 → 去充值。
4. 充值返回 → GET 验证原订单 → 恢复 `pointsUsed`、`payAmount` 和收货快照。
5. 验证接口 500 或网络失败 → 保留会话 → 阻止新建 → 手动重新验证。
6. 支付密码错误 → 不清除原订单。
7. 支付成功 → 清除会话 → 跳订单详情。
8. 购物车多商品待支付字典不被本次改动破坏。

明确回答：

- 是否可能创建第二个订单；
- 是否可能自动支付；
- 是否可能因隐藏地址字段而无法重新支付；
- 是否可能把原订单 `payAmount` 替换为页面 `finalPrice`。
