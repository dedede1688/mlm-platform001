## 三、业务核验

### 3.1 后端

独立确认：

1. 查询条件真实使用 `status: { in: ['pending', 'approved'] }`；
2. pending / approved 均拒绝；
3. rejected 允许重新申请；
4. 拒绝路径不 create、不写日志、不发通知；
5. 未登录、订单不存在、非本人、非法订单状态、空原因保持原行为；
6. 退款创建、金额、通知调用和成功响应未越界修改；
7. 未改退款审核、完成、资金服务和数据库。

### 3.2 前端

独立确认：

1. 金额明细后只有一个售后服务区域；
2. 页面不存在旧满宽橙色退款按钮；
3. 页面不存在重复“退款申请审核中”横条；
4. 无申请只有一个小号“申请退款”；
5. pending / approved / completed 没有申请按钮；
6. rejected 只有一个“重新申请”；
7. 未知状态不会错误放行；
8. `<details>` / `<summary>` 展开结构真实存在；
9. 退款原因、说明、备注、凭证和申请时间没有丢失；
10. 桌面和移动响应式规则合理，无嵌套卡片。
