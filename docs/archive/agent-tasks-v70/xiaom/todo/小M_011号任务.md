```powershell

```

确认业务差异恰好为猫爪 011 号任务规定的 15 个文件，没有其他业务、测试、配置、数据库、工作流、依赖或锁文件改动。

旧文档、任务文件和 `vercel_ls.txt` 不属于本任务，不得修改或清理。

### 3. 两个高风险点

#### `src/app/api/admin/users/route.ts`

确认仍通过解构排除：

```text
orders
referrerId
```

即使变量改成 `_orders` 和 `_referrerId`，也不能把这两个字段展开进返回数据。

#### `src/components/layout/Header.tsx`

确认只从组件解构中移除未使用的 `syncFromStorage`，以下真实调用仍存在：

```typescript
useAuthStore.getState().syncFromStorage()
```

不得破坏登录状态同步和未读通知逻辑。

### 4. 代码规范检查

运行：

```powershell
.\node_modules\.bin\next.cmd lint
```

确认：

- 退出码为 0。
- `Error（阻断错误）`为 0。
- 可以保留原有 `Warning（警告）`。
- 没有修改规则或添加忽略注释。

### 5. 类型检查

运行：

```powershell
npx tsc --noEmit -p tsconfig.typecheck.json
```

确认退出码 0、0 错误。

### 6. 空白和范围检查

运行：

```powershell
git diff --check
git status --short
```

Windows 换行提示可以说明，但不能有真实空白错误。
