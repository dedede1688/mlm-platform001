### 8.3 状态串台复现路径（任务文件四.6 重点场景）

**场景：商品 A 余额不足 → 关闭弹窗 → 打开商品 B 弹窗**

| 步骤 | 操作 | 状态变化 |
|------|------|----------|
| 1 | 点商品 A "立即购买" | `checkoutItem = A` |
| 2 | 点确认 → 余额不足 | `pendingOrder = {cartItemId:A, orderId:O1}`、`shortage = 50`、`showEarningsTransfer = true` |
| 3 | 点 X 关闭弹窗 | `handleCheckoutClose` → `setCheckoutItem(null)`；**`shortage` 仍 = 50**；**`showEarningsTransfer` 仍 = true**；`pendingOrder` 仍 = A 的 |
| 4 | 点商品 B "立即购买" | `handleBuyNow(B)` → `setCheckoutItem(B)`；**`shortage` 仍 = 50**；**`showEarningsTransfer` 仍 = true**；`pendingOrder.cartItemId = A` |
| 5 | 商品 B 弹窗显示 | **`shortage = 50 > 0` → "购物余额不足，还差 ¥50.00"** ❌（实际 B 的差额可能不是 50）|
| 6 | 同时 | **`EarningsTransferModal` 因为 `open=true` 仍覆盖在 B 弹窗之上** ❌（这是 A 触发的 modal） |
| 7 | B 弹窗的 hasPendingOrder 判断 | `pendingOrder.cartItemId(A) === B.id` = false → `hasPendingOrder = false` → 按钮变"确认下单" ✓ |

### 8.4 严重性评估

- **资金层面**：未发现自动支付 / 重复扣款 / 资金串台（用户必须主动点"确认下单"才会创建 B 订单）
