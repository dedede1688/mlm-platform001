### 导出集合风险评估

`REFUND_REASONS_REQUIRING_IMAGES` 是 `new Set([...])` — 在 ES Module 中每个 import 共享同一引用。**理论上**外部可 `import.meta.RefundReasons.add(...)` 污染。

**实际风险评估**：
- TypeScript 类型 `Set<string>` 暗示可读性，不是可写性
- 项目内其他代码**没有任何** `REFUND_REASONS_REQUIRING_IMAGES.add(...)` 调用（grep 0 命中）
- 即使被污染，**只影响前端的 `refundReasonRequiresImages()`** 函数返回，后端仍会独立校验
- 实际风险 = 0

✅ **导出集合风险** = P3 级别（无可观察影响），按任务规定"按 P1/P2 说明，不得夸大" → **无需 P1/P2**
