- **影响**：用户打开弹窗看到一个空输入框 + 灰按钮，不理解为什么。
- **建议**（后续 P2 派单）：弹窗内 `earningsAvailable <= 0` 时显示提示卡："暂无可转入收益，请先去充值或领取分红"。

### P2-1 支付密码用 `window.prompt`

- 订单详情页 L168 仍用 `window.prompt` 输入支付密码，不是最终 UI 方案。

### P2-2 兜底 shortage 计算的时间差

- L193：`const shortage = data.data?.shortage ?? (order.payAmount - userBalance)`
- 兜底路径触发条件是"旧版后端无 `code` 字段、仅靠 `error.includes('余额不足')` 命中"。
- 此时 `userBalance` 来自页面加载时一次 `/api/users/me` 调用，可能与支付时实际余额有 1-2 秒时间差。
- **影响**：建议金额可能略偏（最多几元）。
- **缓解**：弹窗层 L38 会再次 `Math.min(initialAmount, earningsAvailable)` 兜底；后端 `earnings-transfer` 接口会做最终校验。

### P2-3 弹窗 useEffect 依赖变化

- 弹窗 L46 useEffect 依赖从 `[open]` 变成 `[open, initialAmount, earningsAvailable]`。
- **不触发死循环**：父级只在 `setShowTransferModal(true)` 同一 tick 写入 `initialAmount`，无 setState 副作用。
- 依赖里 `earningsAvailable` 来自 props，父级在 `fetchUserBalance` 后才更新，弹窗只在"打开"瞬间读一次最新值并填入 input，符合预期。

### P2-4 `/payment/order/[orderId]` 支付页未覆盖

- 胡子哥在猫爪结果文件 9.3 也确认了后续可扩展。

### P2-5 转入失败弹窗不关闭 + 转入成功弹窗关闭

- 弹窗 L100-103：失败 → `setErrorMsg`，**不关闭** ✅
