### 🟡 P1-B：按钮正则可能跨 `</button>` 标签（**脆性，当前 config 不会触发**）

**实际正则**（场景 1-4 L370, 382, 394, 406）：
```typescript
const buttonMatch = html.match(/<button[^>]*disabled[^>]*>[\s\S]*?重新确认支付[\s\S]*?<\/button>/)
```

**任务 10.0.4 原文**：
> "该正则可能从较早的 disabled 按钮开始，跨过一个或多个 </button> 后才命中'重新确认支付'。必须证明它只匹配目标按钮；如果没有先枚举完整 <button>...</button> 再按文字选择目标按钮，列为 P1 测试可信度问题。"

**结构分析**：
- `<button[^>]*disabled[^>]*>` ← 匹配第一个 disabled button 标签
- `[\s\S]*?` ← 非贪婪任意字符（**但仍能跨 `</button>` 标签**）
- `重新确认支付` ← 字面文本
- `[\s\S]*?` ← 非贪婪任意字符
- `<\/button>` ← 闭合标签

**潜在匹配问题**：
- HTML 中有多个 disabled 按钮（"收益转入余额"、"去充值"、以及 "重新确认支付"）
- 正则从第一个 disabled button 开始，跨过 `</button>` 标签，最终找到 "重新确认支付" 文字
- 测试断言 `not.toBeNull()` 通过，但**实际目标按钮可能不是 disabled**

**当前 config 是否触发**：
- `baseProps` 中**没有** `earningsAvailable` 和 `onGoRecharge`
- 所以 "收益转入余额" 和 "去充值" 按钮**不会渲染**
- 当前测试 config 中 disabled 按钮**只有 "重新确认支付"**
- 正则工作正确

**但脆性问题**：
- 如果未来 baseProps 添加 `earningsAvailable: 100`，正则会**错误地**通过
- 测试不会捕获 "重新确认支付" 按钮实际是 enabled 但被其他 disabled 按钮的"溢出匹配"假通过

**建议修复方案**：

```typescript
// 先枚举所有 <button>...</button>，再按文字选目标
const allButtons = [...html.matchAll(/<button[^>]*>([\s\S]*?)<\/button>/g)]
const confirmButton = allButtons.find(b => b[1].includes('重新确认支付'))
expect(confirmButton, 'confirm button must exist').toBeDefined()
const isDisabled = confirmButton![0].includes('disabled')
expect(isDisabled, 'confirm button must be disabled').toBe(true)
```

**等级**：**P1 测试可信度脆性**（任务 10.0.4 明确 P1；当前 config 下不触发但脆性）

## 十一、按钮状态精确断言证据

### 11.1 当前正则结构

```typescript
/<button[^>]*disabled[^>]*>[\s\S]*?重新确认支付[\s\S]*?<\/button>/
```

**结构分解**：
1. `<button[^>]*disabled[^>]*>` — 匹配第一个带 `disabled` 的 button
2. `[\s\S]*?` — 非贪婪任意字符（**但能跨 `</button>` 标签**）
3. `重新确认支付` — 字面文本
4. `[\s\S]*?` — 非贪婪任意字符
5. `<\/button>` — 闭合标签

### 11.2 任务 10.0.4 警告

> "该正则可能从较早的 disabled 按钮开始，跨过一个或多个 </button> 后才命中'重新确认支付'。必须证明它只匹配目标按钮"

### 11.3 当前 config 是否触发

**CheckoutDialog.tsx 中 disabled 按钮清单**（hasPendingOrder=true, submitting=false, baseProps 中无 earningsAvailable）：
1. `onClose` 按钮：未 disabled
2. `<select disabled={submitting}>`：submitting=false → **不渲染 disabled**
3. "收益转入余额" 按钮（L438）：`earningsAvailable > 0` 才渲染，baseProps 未提供 → **不渲染**
4. "去充值" 按钮（L447）：同上 → **不渲染**
5. "显示/隐藏支付密码" 按钮（L473）：未 disabled
6. "确认订单/重新确认支付" 按钮（L498）：`hasPendingOrder=true && !hasCompleteLockedShipping` → **disabled**

**当前 config**：只有 1 个 disabled button（目标按钮），正则工作正确 ✅

**但脆性场景**：
- 未来 baseProps 添加 `earningsAvailable: 100, onGoRecharge: () => {}, onOpenEarningsTransfer: () => {}`
