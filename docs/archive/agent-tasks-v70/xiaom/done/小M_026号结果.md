### 🔴 P0-2：Number(null/''/false) 被误判为合法 0 缺口（小酷疑点 B，**未闭合**）

**真实代码**（`src/app/products/[id]/page.tsx:478-491`）：

```typescript
const apiShortage = Number(verifyErr.data?.shortage)        // ← Number 强制转换
const paymentAmount = currentPayAmount
let actualShortage: number | null = null

if (Number.isFinite(apiShortage) && apiShortage >= 0) {
  actualShortage = apiShortage                              // ← null/''/false 全部走到这里
} else {
  const calculatedShortage = calculatePendingShortage(paymentAmount, user.balance)
  if (calculatedShortage === null) {
    setRestoreStatus('validation_error')
    setRestoreError('资金数据异常，请重新验证')
    return null
  }
  actualShortage = calculatedShortage
}
```

**小酷疑点 B 真实推演**（已用 Node 验证）：

```text
Number(null) = 0       Number.isFinite(0) = true      0 >= 0 = true    → 当作 0
Number('')   = 0       Number.isFinite(0) = true      0 >= 0 = true    → 当作 0
Number(false) = 0      Number.isFinite(0) = true      0 >= 0 = true    → 当作 0
Number(true)  = 1      Number.isFinite(1) = true      1 >= 0 = true    → 当作 1（更糟！）
Number(undefined) = NaN   Number.isFinite(NaN) = false                       → 走 else 分支
Number('-1') = -1     Number.isFinite(-1) = true     -1 >= 0 = false  → 走 else 分支
```

**真实危害**：

```text
场景 A: 后端 bug 返回 data.shortage = null
─────────────────────────────────────────
1. 余额不足响应：{ code: 'INSUFFICIENT_BALANCE', data: { shortage: null } }
2. apiShortage = Number(null) = 0
3. isFinite(0) && 0 >= 0 → true → actualShortage = 0
4. setPendingPayment({...shortage: 0})
5. persistPendingPayment(orderId, 0) → sessionStorage 写入 shortage: 0
6. user.earningsAvailable > 0 → setShowEarningsTransfer(true) 打开收益转余额弹窗
7. 用户看到 "购物余额不足，还差 ¥0.00"  ← 显示 0 缺口，但实际支付仍会失败
8. 用户困惑：明明显示 0，为什么还说不足？

场景 B: 后端返回 data.shortage = '' (空字符串)
─────────────────────────────────────────
- 完全同场景 A

场景 C: 后端返回 data.shortage = false (布尔)
─────────────────────────────────────────
- 完全同场景 A

场景 D: 后端返回 data.shortage = true (布尔)
─────────────────────────────────────────
- apiShortage = Number(true) = 1
- isFinite(1) && 1 >= 0 → true → actualShortage = 1
- sessionStorage 写入 shortage: 1 ← 错误地把布尔当数字
```

**任务 7.2 明确要求**：
> "负数、NaN、Infinity、null、空字符串等非法 `data.shortage` 不得被转换成 0"

**当前实现**：`null/''/false` **全部被转成 0**，**直接违反任务 7.2**。

**任务 7.2 期望行为**：
- `data.shortage = null` → **不**走 `Number.isFinite && >= 0` 分支 → 走 `calculatePendingShortage` 兜底
- `calculatePendingShortage(payAmount, user.balance)` 正常返回实际缺口 → 走 else 分支更新
- 兜底返回 null → 进入 `validation_error`（不持久化 0）

**当前实现不满足**：`null/''/false` 都在第一道关卡被误判为合法 0，根本走不到兜底。

**等级**：**P0**（违反任务核心要求 + 用户界面显示错误信息 + 持久化错误数据到 sessionStorage）

### 7.1 `extractBlock` 找不到起止标记时是否真实失败

```typescript
function extractBlock(src: string, startMarker: string, endMarker: string): string {
  const startIdx = src.indexOf(startMarker)
  if (startIdx === -1) {
    throw new Error(`extractBlock: start marker "${startMarker}" not found in source`)
  }
  const afterStart = startIdx + startMarker.length
  const endIdx = src.indexOf(endMarker, afterStart)
  if (endIdx === -1) {
    throw new Error(`extractBlock: end marker "${endMarker}" not found after start marker "${startMarker}"`)
  }
  return src.slice(startIdx, endIdx + endMarker.length)
}
```

```typescript
if (typeof rawShortage === 'number' && Number.isFinite(rawShortage) && rawShortage >= 0) {
  actualShortage = rawShortage
} else {
  const calculatedShortage = calculatePendingShortage(paymentAmount, user.balance)
  if (calculatedShortage === null) {
    setRestoreStatus('validation_error')
    setRestoreError('资金数据异常，请重新验证')
    return null
  }
  actualShortage = calculatedShortage
}
```

**关键**：先 `typeof === 'number'` 验证，再 `Number.isFinite`，最后 `>= 0`。这样 `null/''/false/undefined` 都不会走第一道关卡。
