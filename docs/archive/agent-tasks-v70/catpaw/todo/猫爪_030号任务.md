### 7.2 渲染辅助函数

建立类似：

```typescript
function renderPendingDialog(
  shipping: CheckoutLockedShipping | null
): string {
  return renderToStaticMarkup(
    React.createElement(CheckoutDialog, {
      open: true,
      product: {
        id: 'product-1',
        name: '测试商品',
        memberPrice: 500,
      },
      onClose: vi.fn(),
      onConfirm: vi.fn().mockResolvedValue(null),
      hasPendingOrder: true,
      pendingOrderShipping: shipping,
    })
  )
}
```

### 7.3 精确提取确认按钮

从渲染 HTML 中提取包含“重新确认支付”的真实 `<button>...</button>`：

```typescript
function getConfirmButton(html: string): string {
  const buttons = [...html.matchAll(/<button\b[^>]*>[\s\S]*?<\/button>/g)]
  const button = buttons.find((match) => match[0].includes('重新确认支付'))
  expect(button, '未找到重新确认支付按钮').toBeDefined()
  return button![0]
}
```

不得通过全文是否存在 `disabled` 来判断确认按钮状态。

### 7.4 五个独立场景

场景 1：`pendingOrderShipping=null`  
场景 2：姓名为 `'   '`，手机和地址完整  
场景 3：手机为 `'   '`，姓名和地址完整  
场景 4：地址为 `'   '`，姓名和手机完整  
场景 5：三字段完整，建议使用 `' 张三 '`、`' 13800138001 '`、`' 广东省广州市测试路 1 号 '`

前 4 个场景必须分别断言：

```text
包含：订单收货信息缺失，请联系客服
不包含：订单已创建，收货信息以首次提交为准
不包含：选择收货地址
重新确认支付按钮包含 disabled 属性
```

第 5 个场景必须断言：

```text
包含：订单已创建，收货信息以首次提交为准
包含：张三
包含：13800138001
包含：广东省广州市测试路 1 号
不包含：订单收货信息缺失，请联系客服
重新确认支付按钮不包含 disabled 属性
```

React 服务端输出的布尔属性通常表现为 `disabled=""`。断言允许兼容标准 HTML 布尔属性格式，但必须限定到 `getConfirmButton(html)` 返回的按钮标签。
