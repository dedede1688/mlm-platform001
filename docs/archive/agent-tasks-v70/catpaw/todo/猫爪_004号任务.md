# 猫爪_004号任务：会员详情改标签页 + 资金账户集中展示和操作

## 一、任务背景

胡子哥实测发现：

1. 后台「会员详情」弹窗现在内容太长，需要一直向下滚动。
2. 「收益账户」虽然已经显示出来，但资金操作区域在很下面，不直观。
3. 胡子哥希望会员详情像浏览器网页一样，用「标签页」切换不同内容。

本任务目标是优化后台「会员详情」弹窗结构，把长滚动改成标签页，并把余额、收益和资金调整操作集中放到「资金账户」标签里。


## 三、任务目标

只修改后台会员管理页面：

```text
src/app/admin/users/page.tsx
```

实现：

1. 「会员详情」弹窗改成标签页切换。
2. 默认打开「基本资料」标签。
3. 新增「资金账户」标签，集中展示余额账户、收益账户、资金调整操作。
4. 原来已有的「可用收益」增加操作，放到「资金账户」标签里更容易找到。
5. 保留现有经营统计、推荐关系、直推列表等内容，但改为对应标签下展示。

## 六、标签页设计

会员详情弹窗顶部标题下方增加标签页，建议标签如下：

| 标签 | 内容 |
|---|---|
| 基本资料 | 手机号、昵称、等级、状态、积分等基础字段 |
| 资金账户 | 余额账户、收益账户、资金调整操作 |
| 经营统计 | 原「经营统计」内容 |
| 推荐关系 | 原「推荐关系」内容 |
| 直推列表 | 原「直推列表」内容 |

如果页面里还有其他详情内容，请按语义放进合适标签，不要删除原有信息。

## 七、资金账户标签要求

「资金账户」标签必须包含以下内容。

### 1. 余额账户

展示：

- 余额
- 冻结余额

如果页面已有积分相关资金字段，也可以放在这里，但不要破坏原展示。

### 2. 收益账户

展示：

| 字段 | 中文展示 |
|---|---|
| `earningsAvailable` | 可用收益 |
| `earningsFrozen` | 冻结收益 |
| `earningsPending` | 待结算收益 |
| `earningsVoided` | 作废收益 |

所有金额字段都要用 `?? 0` 兜底。

### 3. 资金调整操作

把现有「资金调整」区域放在「资金账户」标签中。

资金调整类型必须继续支持：

- 余额
- 冻结余额
- 可用收益

其中「可用收益」提交给后端的类型必须仍然是：

```text
earnings_add
```

当类型为 `earnings_add` 时：

- 只允许输入正数。
- 0 或负数必须前端拦截。
- 提示文案继续使用或保持类似：

```text
本次只允许增加可用收益
```

## 八、交互要求

1. 打开会员详情时，默认显示「基本资料」。
2. 点击不同标签时，只切换弹窗内容，不关闭弹窗。
3. 切换标签时，不要丢失当前会员详情数据。
4. 在「资金账户」里做资金调整成功后，沿用现有刷新逻辑。
5. 如果详情弹窗仍打开，刷新后应能看到最新收益数据。
6. 标签按钮要明显区分当前选中状态。
7. 标签过多时，允许横向滚动或自动换行，但不能挤压错乱。

## 九、样式要求

1. 保持当前后台页面风格。
2. 不要做大面积视觉重构。
3. 卡片圆角、边框、间距与当前页面一致。
4. 标签页样式应清晰、紧凑、方便点击。
5. 弹窗底部「关闭」按钮保留。

## 十、验收标准

完成后必须满足：

1. 会员详情弹窗不再是所有内容一条长滚动堆叠。
2. 能看到标签页：基本资料、资金账户、经营统计、推荐关系、直推列表。
3. 点「资金账户」能直接看到余额账户、收益账户、资金调整。
4. 收益账户能看到可用收益、冻结收益、待结算收益、作废收益。
5. 资金调整里能选择「可用收益」。
6. 「可用收益」提交类型仍是 `earnings_add`。
7. 输入 0 或负数时仍然会前端拦截。
8. 未新增收益扣减、作废收益、余额转回收益。
9. 未修改后端、数据库、支付、提现、订单逻辑。
