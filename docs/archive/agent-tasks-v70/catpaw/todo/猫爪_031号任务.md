## 六、P1-2：精确定位“重新确认支付”按钮

### 6.1 删除跨标签正则

删除所有类似：

```typescript
/<button[^>]*disabled[^>]*>[\s\S]*?重新确认支付[\s\S]*?<\/button>/
/<button[^>]*>[\s\S]*?重新确认支付[\s\S]*?<\/button>/
```

这类正则可能从前一个按钮开始跨越 `</button>`，再命中后面的“重新确认支付”。

### 6.2 新增唯一按钮提取辅助函数

使用完整按钮枚举，再按文字筛选：

```typescript
function getButtonByText(html: string, text: string): string {
  const buttons = [...html.matchAll(/<button\b[^>]*>[\s\S]*?<\/button>/g)]
    .map((match) => match[0])
  const matchingButtons = buttons.filter((button) => button.includes(text))

  expect(matchingButtons, `应唯一找到按钮：${text}`).toHaveLength(1)
  return matchingButtons[0]
}
```

要求：

1. 先枚举每个完整 `<button>...</button>`；
2. 再按“重新确认支付”筛选；
3. 必须断言恰好 1 个；
4. 找不到或找到多个都必须失败；
5. 禁止 `if (match)` 静默跳过；
6. 禁止 `A || B` 任一路径通过。

### 6.3 精确断言 disabled

4 个无效快照场景：

```typescript
const confirmButton = getButtonByText(html, '重新确认支付')
expect(confirmButton).toMatch(/\sdisabled(?:=""|(?=\s|>))/)
```

完整快照场景：

```typescript
const confirmButton = getButtonByText(html, '重新确认支付')
expect(confirmButton).not.toMatch(/\sdisabled(?:=""|(?=\s|>))/)
```

允许使用等价的 HTML 布尔属性正则，但必须只检查辅助函数返回的目标按钮。
