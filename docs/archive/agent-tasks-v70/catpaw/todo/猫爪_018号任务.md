## 二、背景

当前结算弹窗中的“忘记密码？去修改”会进入支付密码修改页，但修改页仍要求输入旧支付密码，忘记密码的用户无法完成操作。

项目已有能力：

- `users.paymentPasswordHash` 允许为空。
- `/dashboard/payment-password` 会通过 `/api/users/me` 的 `hasPaymentPassword` 判断进入首次设置或修改模式。
- `/api/user/payment-password/set` 可以为未设置支付密码的用户保存新的 6 位支付密码。
- 当前短信发送仍是模拟能力，本任务不做短信验证码重置。

胡子老师已确认采用“后台人工核验后清除旧支付密码，用户自行设置新密码”的方案。

## 三、目标

实现完整闭环：

1. 用户在结算弹窗点击“忘记支付密码”，看到联系客服说明弹窗。
2. 超级管理员在会员详情“资金账户”标签查看支付密码状态。
3. 超级管理员填写原因和手机号后 4 位后清除旧支付密码。
4. 系统写操作日志并发送站内通知。
5. 用户进入现有支付密码页面自行设置新的 6 位支付密码。
6. 用户手动重新支付，系统不得自动扣款。

## 四、强制业务规则

### 4.1 权限

- 后端必须使用：

```typescript
verifyPermission(request, ['super_admin'])
```

- 中间件允许 `support_admin` 进入 `/api/admin/users/*` 不代表接口可以授权客服管理员；接口必须再次限制为 `super_admin`。
- 前端只有 `userRole === 'super_admin'` 时显示“重置支付密码”按钮。
- 其他可查看会员详情的管理员只能看到“已设置/未设置”状态。

### 4.2 二次确认

- `reason` 必须是字符串，`trim()` 后不少于 5 个字。
- `phoneSuffix` 必须是恰好 4 位数字。
- `phoneSuffix` 必须等于目标用户手机号后 4 位。
- 前端校验只是体验优化，后端必须重复全部校验。

### 4.3 重置行为

- 管理员只清除 `paymentPasswordHash`。
- 管理员不能输入、查看或生成用户的新支付密码。
- 用户不存在或状态为 `deleted` 时返回 404。
- 用户本来就未设置支付密码时返回 400，不重复执行。
- 使用条件更新防并发重复重置，建议：

```typescript
const updated = await prisma.user.updateMany({
  where: {
    id,
    paymentPasswordHash: { not: null },
  },
  data: { paymentPasswordHash: null },
})
```

- `updated.count !== 1` 时不得报告重置成功。
- 不修改 `passwordHash`、余额、冻结余额、消费余额、任何收益字段、积分、订单和角色。

### 4.4 日志和通知

- 操作日志动作使用现有 `logOperation` 对象参数风格。
- 日志记录管理员、目标用户、原因、网络地址和浏览器信息。
- 日志中禁止出现旧支付密码、新支付密码或 `paymentPasswordHash`。
- 新增通知方法 `notifyPaymentPasswordReset`，放在 `OrderNotificationService` 中，方法整体必须有 `try/catch`。
- 路由层必须 `await` 通知方法；通知失败不得让已经成功的密码清除返回 500。
- 通知建议关联：

```typescript
sourceType: 'payment_password'
sourceId: userId
```

### 4.5 用户支付流程

- 已设置支付密码时，把“忘记密码？去修改”替换成“忘记支付密码？”按钮。
- 点击按钮在当前页面打开说明弹窗，不得跳到要求旧密码的修改页面。
- 弹窗文案明确要求联系客服核验身份。
- “联系客服”进入 `/help`（帮助中心），“取消”关闭说明弹窗。
- 未设置支付密码时显示“立即设置”，进入 `/dashboard/payment-password`。
- 本任务不得在重置或设置密码后自动调用支付接口。

## 五、接口设计

新增：

```text
POST /api/admin/users/[id]/payment-password/reset
```

请求体：

```json
{
  "reason": "用户本人联系客服申请重置",
  "phoneSuffix": "8001"
}
```

成功响应建议：

```json
{
  "success": true,
  "data": {
    "hasPaymentPassword": false
  },
  "message": "支付密码已重置，请通知用户重新设置"
}
```

错误状态：

- 400：参数错误、后 4 位不匹配、用户未设置支付密码。
- 401/403：未登录或非超级管理员。
- 404：用户不存在或已删除。
- 409：条件更新未命中，状态已被其他操作改变。
- 500：真实未知异常。
