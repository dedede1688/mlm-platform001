## 4. 防止空订单编号的实现说明

**修复前**（有 bug）：
```typescript
setPendingPayments(prev => ({
  ...prev,
  [checkoutItem.id]: {
    orderId: prev[checkoutItem.id]?.orderId ?? '',  // ← 空字符串兜底
    shortage: remainingShortage,
  },
}))
```

问题：如果 `prev[checkoutItem.id]` 不存在，会创建 `{ orderId: '', shortage: ... }` 伪记录，下一次确认支付时可能请求 `/api/orders//verify-payment`。

**修复后**（防御性）：
```typescript
setPendingPayments(prev => {
  const current = prev[checkoutItem.id]
  if (!current) return prev  // ← 记录不存在时直接返回原状态，不新增任何键

  return {
    ...prev,
    [checkoutItem.id]: {
      ...current,            // ← 保留真实 orderId
      shortage: remainingShortage,  // ← 只更新 shortage
    },
  }
})
```

关键保障：
- 记录不存在时 `return prev`，不新增任何键 → 不会产生空 `orderId` 伪记录
- 记录存在时 `...current` 展开保留真实 `orderId` → 不依赖空字符串兜底
- `...prev` 展开保留其他商品记录 → 不影响其他商品
- 不包含 `verify-payment` / `handleCheckoutConfirm` → 不自动支付
