## 3. 具体做了什么

### 3.1 EarningsTransferModal（收益转余额弹窗）增强

- 新增可选 prop `initialAmount?: number`
- 弹窗打开时，如果传入了 `initialAmount`，自动填入建议转入金额（`min(initialAmount, earningsAvailable)`）
- 不传 `initialAmount` 时行为不变（空白输入），完全向后兼容

### 3.2 订单详情页改动

1. **读取用户资金信息**：页面加载时同时调 `GET /api/users/me`，拿到 `balance`（购物余额）和 `earningsAvailable`（可用收益），带 `Authorization: Bearer token` 鉴权头。

2. **捕获余额不足错误**：`handlePay` 中调用 `POST /api/orders/[id]/verify-payment` 后，主判断 `data.code === 'INSUFFICIENT_BALANCE'`，兜底字符串判断 `data.error.includes('余额不足')`（防旧版后端）。

3. **打开弹窗**：余额不足时计算建议转入金额 `min(shortage, earningsAvailable)`，传入 `initialAmount` 打开 `EarningsTransferModal`。

4. **转入成功后**：
   - 刷新用户资金信息（重新调 `/api/users/me`）
   - toast 提示「余额已补充，请重新点击支付」
   - **不自动重新支付**（支付需要支付密码，不能绕过用户确认）

5. **其他错误路径不变**：
   - 支付成功路径：toast 成功 + 刷新订单
   - 支付密码错误：走 `else` 分支 toast 错误
   - 网络/其他错误：走 `catch` 分支 toast 网络错误
