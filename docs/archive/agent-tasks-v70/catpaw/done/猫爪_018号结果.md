## 1. 任务概述

实现"忘记支付密码的后台人工重置流程"，包括：
- 超级管理员后台重置用户支付密码（清除 `paymentPasswordHash`）

## 6. 关键代码变更摘要

### 6.1 支付密码重置接口（新增）

```typescript
// src/app/api/admin/users/[id]/payment-password/reset/route.ts
// 显式校验必须是超级管理员（防御性编程）
if (admin.role !== 'super_admin') {
  return errorResponse('权限不足，仅超级管理员可执行此操作', 403)
}

// 条件更新 - 防止并发重复重置
const updated = await prisma.user.updateMany({
  where: { id: userId, paymentPasswordHash: { not: null } },
  data: { paymentPasswordHash: null },
})
```

### 6.2 会员列表接口安全过滤

```typescript
// src/app/api/admin/users/route.ts
const result: Record<string, unknown> = {
  ...rest,
  hasPaymentPassword: !!_paymentHash,  // 只返回布尔值

}
delete result.passwordHash  // 防御性删除
```

## 7. 尚未执行的人工浏览器验收项目

以下项目需胡子老师在真实浏览器中验收：

### 后台验收（admin 账号登录）
1. **会员列表显示支付密码状态**：`/admin/users` 列表应显示"已设置/未设置"标签
2. **点击会员详情 → 支付安全区域**：应看到"支付密码状态"和"重置支付密码"按钮
3. **重置支付密码流程**：输入原因 + 手机号后 4 位 → 成功提示 → 状态变为"未设置"
4. **非超管账号**：不应看到"重置支付密码"按钮

### 用户端验收（普通用户账号登录）
5. **已设置支付密码用户**：结算弹窗显示"忘记支付密码？查看帮助"
6. **点击查看帮助**：弹出说明弹窗，包含"请联系客服核验身份后重置"
7. **未设置支付密码用户**：结算弹窗显示"未设置？立即设置"，链接到 `/dashboard/payment-password`
8. **关闭说明弹窗**：不影响结算弹窗，不触发任何支付动作

### 通知验收
9. **重置成功后**：用户端铃铛应收到"支付密码已重置"通知
