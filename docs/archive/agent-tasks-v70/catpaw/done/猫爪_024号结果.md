## 8. 数据流

### 首次创建
1. 用户点"立即购买" → 打开 CheckoutDialog
2. 填写收货信息 + 支付密码 → `handleCheckoutConfirm`
3. 创建订单 → 保存 `pendingPayment`（内存）+ `persistPendingPayment`（sessionStorage）
4. 调用 `verify-payment`

### 余额不足
5. `verify-payment` 返回 `INSUFFICIENT_BALANCE`
6. 计算 `actualShortage`（优先用 `verifyErr.data.shortage`，兜底用 `calculatePendingShortage`）
7. 更新 `pendingPayment.shortage` + `persistPendingPayment`
8. `earningsAvailable > 0` → 打开收益弹窗；否则提示充值

### 收益转入
9. `EarningsTransferModal` 的 `initialAmount = min(shortage, earningsAvailable)`
10. 转入成功 → `handleEarningsTransferSuccess`
11. 刷新 `/api/users/me` → 用 `pendingPayment.payAmount - latestUser.balance` 重算缺口
12. 不调用支付接口

### 充值恢复
13. 去充值前 `saveProductPendingPayment` → 失败则不跳转
14. 充值返回 → `restorePendingPayment(false)` 自动触发
15. GET `/api/orders/${orderId}` + Bearer token → 校验 pending + 商品匹配
16. 恢复 `pointsUsed` + `payAmount` + 收货快照

### 重新验证
17. 500/网络错误 → `validation_error` + "重新验证"按钮
18. 按钮调用 `restorePendingPayment(true)` 绕过去重

### 支付成功
19. `clearProductPendingPayment` + 清空 `pendingPayment` + 关闭弹窗 + 跳转订单详情
