## 4. 新状态结构

```typescript
// v015: 按商品索引的待支付记录（orderId + shortage 一体保存）
interface PendingCartPayment {
  orderId: string
  shortage: number
}

const [pendingPayments, setPendingPayments] = useState<Record<string, PendingCartPayment>>({})

// 渲染期间派生当前商品记录
const currentPendingPayment = checkoutItem
  ? pendingPayments[checkoutItem.id]
  : undefined

const currentShortage = currentPendingPayment?.shortage ?? 0
```

旧状态已移除：
- `useState<PendingCartOrder | null>` ❌
- `useState<number>` (shortage) ❌
- `setShortage()` ❌
- `setPendingOrder()` ❌

## 5. A→B→A 状态推演

| 步骤 | 操作 | pendingPayments 状态 | 行为 |
|------|------|----------------------|------|
| 1 | 打开 A，创建 orderA，余额不足 shortageA=100 | `{cartA: {orderA, 100}}` | 写入 A 记录 |
| 2 | 关闭 A | `{cartA: {orderA, 100}}` | 不删除任何记录 |
| 3 | 打开 B | `{cartA: {orderA, 100}}` | currentShortage=0（B 无记录） |
| 4 | 创建 orderB，余额不足 shortageB=200 | `{cartA: {orderA, 100}, cartB: {orderB, 200}}` | 不覆盖 A |
| 5 | 关闭 B，再打开 A | `{cartA: {orderA, 100}, cartB: {orderB, 200}}` | currentShortage=100（A 记录仍在） |
| 6 | 再次确认 A | `{cartA: {orderA, 100}, cartB: {orderB, 200}}` | 跳过 POST /api/orders，直接用 orderA |
| 7 | A 支付成功 | `{cartB: {orderB, 200}}` | 只删除 A，B 保留 |
| 8 | 收益转入成功 | 只更新当前商品 shortage | 不自动支付 |

## 6. 新订单写入是否保留其他商品记录

✅ **保留**。使用函数式更新展开旧记录：

```typescript
setPendingPayments(prev => ({
  ...prev,
  [checkoutItem.id]: {
    orderId: confirmedOrderId,
    shortage: 0,
  },
}))
```

测试断言：`expect(block).toMatch(/setPendingPayments\(\s*prev\s*=>/)` + `expect(block).toMatch(/\.\.\.prev/)`

## 7. 支付成功是否只删除当前商品记录

✅ **只删除当前商品**。使用 `delete next[checkoutItem.id]`，不清空整张表：

```typescript
setPendingPayments(prev => {
  const next = { ...prev }
  delete next[checkoutItem.id]
  return next
})
```

测试断言：
- `expect(block).toMatch(/delete\s+next\[checkoutItem\.id\]/)`
- `expect(block).not.toMatch(/setPendingPayments\(\s*\{\s*\}\s*\)/)`

## 8. 收益转入是否只更新当前商品缺口

✅ **只更新当前商品**。从 `pendingPayments[checkoutItem.id]` 读取旧缺口，计算剩余差额后只写回当前商品：

```typescript
const existingShortage = pendingPayments[checkoutItem.id]?.shortage ?? 0
const remainingShortage = Math.max(0, existingShortage - transferred)

setPendingPayments(prev => ({
  ...prev,
  [checkoutItem.id]: {
    orderId: prev[checkoutItem.id]?.orderId ?? '',
    shortage: remainingShortage,
  },
}))
```

`checkoutItem` 为空时安全返回，只刷新资金：
```typescript
if (!checkoutItem) {
  if (token) fetchUserInfo(token)
  return
}
```

测试断言：
- `expect(block).toMatch(/setPendingPayments\(\s*prev\s*=>/)`
- `expect(block).toMatch(/checkoutItem\.id/)`
- `expect(block).toMatch(/!checkoutItem|checkoutItem\s*===\s*null/)`
