## 4. P0 通知失败隔离如何修复

### 修复前（有问题）

```typescript
static async notifyEarningsVoid(params: ...) {
  // ❌ 用户查询在 try/catch 之外
  const user = await prisma.user.findUnique(...)
  if (!user) return
  // ...
  await (async () => {
    try {
      // 只有 batch.create 和 sendInApp 在 try/catch 内
    } catch (err) { ... }
  })()  // ❌ 未 await 的 IIFE
}
```

### 修复后

```typescript
static async notifyEarningsVoid(params: ...) {
  try {
    // ✅ 用户查询在 try/catch 内
    const user = await prisma.user.findUnique(...)
    if (!user) return
    // ✅ batch.create 在 try/catch 内
    const b = await prisma.notificationBatch.create(...)
    // ✅ sendInApp 在 try/catch 内
    await sendInApp(...)
  } catch (err) {
    console.error('[v006 notifyEarningsVoid]', { error: String(err), ... })
    logger.error('收益作废通知失败', { error: String(err) })
  }
}
```

- 删除了未 await 的 IIFE 包装
- 整个方法体被 try/catch 覆盖
- 用户查询、批次创建、站内信发送三步都在保护内

## 5. 路由层第二道保护如何实现

```typescript
// src/app/api/admin/users/[id]/balance/route.ts
if (adjustType === 'earnings_void') {
```

- 即使 `notifyEarningsVoid` 内部 try/catch 被绕过（未来代码修改引入 bug），路由层 `.catch()` 仍能兜底

## 7. 并发真实快照如何计算

### 修复前（有问题）

```typescript
// 先读 before
const before = await tx.user.findUnique({ where: { id } })
// ... updateMany ...
// 用 before 伪造更新后值
const after4Field = {
  earningsAvailable: before.earningsAvailable - amount,  // ❌ 伪造
  earningsVoided: before.earningsVoided + amount,         // ❌ 伪造
}
// BalanceRecord 用 before.balance / before.frozenBalance
```

并发等待后 `before` 可能不是本次更新真正发生前的值。

### 修复后

```typescript
// 1. updateMany 成功
const voidResult = await tx.user.updateMany({
  where: { id, earningsAvailable: { gte: amount } },
  data: { earningsAvailable: { decrement: amount }, earningsVoided: { increment: amount } },
})
if (voidResult.count === 0) throw new Error('可用收益不足')

// 2. 读取更新后的真实值
const updated = await tx.user.findUnique({ where: { id } })
if (!updated) throw new Error('用户更新后查询失败')

// 3. 反推本次操作前的真实前值
const actualOld = {
  earningsAvailable: updated.earningsAvailable + amount,
  earningsVoided: updated.earningsVoided - amount,
}

// 4. BalanceRecord 用 updated 的真实值
await tx.balanceRecord.create({
  data: {
    balance: updated.balance,           // ✅ 真实快照
    frozenBalance: updated.frozenBalance, // ✅ 真实快照
    description: `...${format4FieldDelta(before4Field, after4Field)}`,
  },
})

// 5. OperationLog 用 actualOld 和 updated
return { updated, oldValue: actualOld, ... }
```

- `updateMany` 条件不变（`earningsAvailable: { gte: amount }`）
- 流水描述使用反推的真实前值 + 读取的真实后值
- `OperationLog.oldValue` = 反推真实前值
- `OperationLog.newValue` = 数据库读取的真实后值
- 通知参数使用数据库读取的真实后值
