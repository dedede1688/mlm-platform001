## 8. 后端失败路径是否保证不写数据库、不发通知

**是。** 后端在 `validateRefundApplication` 返回 `{ success: false }` 时直接 `return NextResponse.json({ error }, { status: 400 })`，不会执行后续的 `prisma.refundRequest.create` 和 `OrderNotificationService.notifyRefundSubmitted`。测试已验证 `prisma.refundRequest.create` 和 `OrderNotificationService.notifyRefundSubmitted` 在校验失败时调用次数为 0。

## 9. 重新申请是否为空表单

**是。** 页面 `useState<RefundForm>` 初始值为 `{ reason: '', description: '', images: [] }`，不读取任何历史退款记录。测试已验证源码中不含 `refundRequest`、`previousRefund` 等历史数据引用。
