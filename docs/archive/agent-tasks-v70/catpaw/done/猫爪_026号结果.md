## 12 步完整业务链路推演

### 正常链路（lockedShipping 完整）

1. 用户打开商品详情页 → `restorePendingPayment` 从 sessionStorage 恢复
2. 恢复成功 → `restoreStatus='restored'`，`pendingPayment` 含完整 shipping
3. 用户点"立即购买" → `handleBuyNow` → 打开 CheckoutDialog
4. CheckoutDialog：`hasPendingOrder=true, lockedShipping=完整` → 显示锁定摘要
5. 用户输入支付密码 → 点"重新确认支付" → `handleConfirm`
6. handleConfirm：`if (hasPendingOrder)` → lockedShipping 完整 → 跳过地址校验
7. shippingSnapshot = lockedShipping → 调 onConfirm → handleCheckoutConfirm
8. verify-payment → 余额充足 → 支付成功 → clearProductPendingPayment → 跳转订单详情

### 余额不足链路（calculatePendingShortage 返回有效值）

1-5 同上
6. verify-payment → 余额不足 → apiShortage >= 0 → actualShortage = apiShortage
7. persistPendingPayment → 打开收益转余额弹窗
8. 用户转收益 → handleEarningsTransferSuccess → fetchUser 成功 → calculatePendingShortage 返回有效值
9. 更新 pendingPayment.shortage → persistPendingPayment → 关闭弹窗
10. 用户再次确认支付 → verify-payment → 余额充足 → 支付成功

### P0-1 异常链路（lockedShipping=null）

1. 用户打开商品详情页 → restorePendingPayment → 订单存在但 shipping 字段为空
2. restorePendingPayment 内：shipping 三字段任一为空 → clearProductPendingPayment → idle
3. **如果 pendingOrderShipping prop 传入 null**（父组件 pendingPayment.shipping 缺失）
4. CheckoutDialog：`hasPendingOrder=true, lockedShipping=null` → 显示只读错误提示
5. 确认按钮 disabled → 用户无法提交
6. 即使绕过 disabled → handleConfirm：`if (hasPendingOrder)` → `!lockedShipping` → toast "订单收货信息缺失，请联系客服" → return

### P0-2 异常链路（calculatePendingShortage 返回 null）

**restorePendingPayment**：
1. 恢复订单 → fetchUser 成功 → calculatePendingShortage 返回 null
2. setRestoreStatus('validation_error') → setRestoreError('资金数据异常，请重新验证')
3. 不调 setPendingPayment → 不调 persistPendingPayment → 不进入 restored
4. 用户看到验证失败提示 → 点"重新验证"

**handleCheckoutConfirm**：
1. verify-payment → 余额不足 → apiShortage 为 NaN/负数
2. calculatePendingShortage 返回 null
3. setRestoreStatus('validation_error') → setRestoreError('资金数据异常，请重新验证')
4. return null → 不更新 pendingPayment.shortage → 不持久化 → 不打开收益弹窗

**handleEarningsTransferSuccess**：
1. 用户转收益 → fetchUser 返回 null
2. setShowEarningsTransfer(false) → setRestoreStatus('validation_error') → setRestoreError('收益已转入，请重新验证资金信息')
3. return → 不更新 pendingPayment → 不持久化
4. 或者 fetchUser 成功 → calculatePendingShortage 返回 null
5. 同上 → validation_error → 不更新不持久化
